2. "Fehlerkorrektur-Level"
   - Ziel: Wähle das richtige Error-Correction-Level (L, M, Q, H).
   - Spielmechanik: 
     - Szenario: "Code kommt auf ein Plakat, sauberer Druck". -> Level L (Wenig Redundanz, Code wird kleiner).
     - Szenario: "Code kommt auf LKW-Plane, wird dreckig, Logo in der Mitte". -> Level H (30% Redundanz, Code wird komplexer/größer).
     - Messbarkeit: Optimale Wahl für den Einsatzzweck.
   - Lernziel: Trade-off Datendichte vs. Robustheit.
