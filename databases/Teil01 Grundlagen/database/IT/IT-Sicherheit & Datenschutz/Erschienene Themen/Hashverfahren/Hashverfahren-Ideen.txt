1. "Der Kollisions-Tester"
   - Ziel: Erzeuge zwei verschiedene Inputs, die den gleichen Hash haben (theoretisch).
   - Spielmechanik: 
     - MD5 (unsicher). Spieler ändert ein Bit in Datei A. Hash ändert sich komplett (Avalanche Effect).
     - Aber: Findet er eine Kollision? (Bei MD5 heute machbar, im Spiel simuliert).
     - Messbarkeit: Verständnis, dass Hashfunktion eigentlich kollisionsfrei sein sollte.
   - Lernziel: Einweg-Funktion und Kollisionsresistenz.

2. "Integritäts-Check"
   - Ziel: Prüfe, ob die Datei beim Download verändert wurde.
   - Spielmechanik: 
     - Original Hash (Webseite): "A1B2...".
     - Download-Hash berechnen: "A1B2...". -> Ok.
     - Manipulation simuliert (Man-in-the-Middle). Download-Hash: "C3D4...".
     - Spieler muss "Löschen" klicken!
     - Messbarkeit: Erkennung der Manipulation.
   - Lernziel: Hash als digitaler Fingerabdruck.

3. "Passwort-Salting"
   - Ziel: Erschwere Rainbow-Table Attacken.
   - Spielmechanik: 
     - User A PW: "123456". Hash: "X".
     - User B PW: "123456". Hash: "X" (Schlecht!).
     - Aufgabe: Füge Salt hinzu.
     - User A: "123456" + "Salt1" -> Hash "Y".
     - User B: "123456" + "Salt2" -> Hash "Z".
     - Messbarkeit: Eindeutige Hashes trotz gleichem PW.
   - Lernziel: Schutz gespeicherter Passwörter.

4. "Algorithmen-Ranking"
   - Ziel: Sortiere von "Unsicher" zu "Sicher".
   - Spielmechanik: 
     - MD5 -> SHA-1 -> SHA-256 -> SHA-3/BCrypt.
     - Drag & Drop auf Zeitleiste.
     - Messbarkeit: Korrekte historische Einordnung.
   - Lernziel: Kryptografische Standards.

5. "Blockchain-Miner (Mini)"
   - Ziel: Finde eine Nonce, sodass der Hash mit "0000" beginnt.
   - Spielmechanik: 
     - Input: "Transaktion XY".
     - Nonce: 1 -> Hash "8F..."
     - Nonce: 2 -> Hash "7A..."
     - ...
     - Spieler klickt "Auto-Mine". Stoppt bei Hash "0000...".
     - Messbarkeit: Funktionsprinzip Proof-of-Work verstanden.
   - Lernziel: Hash in der Blockchain.
