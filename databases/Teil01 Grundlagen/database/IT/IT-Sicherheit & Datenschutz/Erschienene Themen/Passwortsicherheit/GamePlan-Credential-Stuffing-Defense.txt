4. "Credential-Stuffing-Defense"
   - Ziel: Verhindere, dass ein geleaktes PW überall funktioniert.
   - Spielmechanik: 
     - User verwendet "Schatzi1" für Facebook, Amazon und Firmen-Email.
     - Facebook wird gehackt.
     - Hacker probiert "Schatzi1" bei Amazon.
     - Spieler muss User zwingen: "Einzigartige Passwörter nutzen" (z.B. Passwort-Manager).
     - Messbarkeit: Anzahl der kompromittierten Accounts = 1 (nur Facebook), nicht alle.
   - Lernziel: Password Reuse vermeiden.
