1. "Der Notfall-Adressen-Check"
   - Ziel: Erkenne, wann APIPA greift.
   - Spielmechanik: 
     - Szenario 1: DHCP Server antwortet. IP: 192.168.1.10.
     - Szenario 2: DHCP Server down. IP: 169.254.x.y (APIPA).
     - Szenario 3: Kabel gezogen. IP: 0.0.0.0 (oder Media disconnected).
     - Messbarkeit: Korrekte Diagnose des Netzwerkstatus anhand der IP.
   - Lernziel: Erkennungsmerkmal 169.254.0.0/16.

2. "APIPA-Insel"
   - Ziel: Verbinde zwei PCs ohne DHCP und ohne manuelle Konfig.
   - Spielmechanik: 
     - PC A und PC B direkt per Kabel verbunden.
     - Beide auf "IP automatisch beziehen".
     - Warten... APIPA greift (Beide kriegen 169.254...).
     - Ping von A nach B funktioniert.
     - Wenn User manuell 192.168... einträgt -> Fehler (Aufgabe war "ohne manuell").
     - Messbarkeit: Erfolgreicher Ping im APIPA-Netz.
   - Lernziel: Zero-Config Networking (Zeroconf).

3. "Der Routing-Fehler"
   - Ziel: Warum geht das Internet nicht?
   - Spielmechanik: 
     - PC hat 169.254.10.10. Gateway ist leer (APIPA vergibt kein Gateway!).
     - User will google.de pingen -> "Destination unreachable".
     - User muss Problem benennen: "Kein Gateway bei APIPA möglich".
     - Messbarkeit: Korrekte Fehleranalyse.
   - Lernziel: Einschränkungen von APIPA (Kein Routing).

4. "Subnetz-Rechner (Klasse B)"
   - Ziel: Berechne die Subnetzmaske für APIPA.
   - Spielmechanik: 
     - APIPA nutzt 169.254.0.0/16.
     - Maske: 255.255.0.0.
     - User tippt 255.255.255.0 -> Falsch.
     - Messbarkeit: Korrekte CIDR-Notation / Maske.
   - Lernziel: Adressbereich der Automatic Private IP Addressing.

5. "Der DHCP-Restaurator"
   - Ziel: Werde APIPA wieder los.
   - Spielmechanik: 
     - PC hängt in APIPA fest.
     - Aufgabe: Repariere den DHCP-Server.
     - Dienst neustarten -> PC sendet Discover -> Bekommt 192.168.x.x.
     - APIPA-Adresse verschwindet.
     - Messbarkeit: PC hat wieder routbare IP.
   - Lernziel: APIPA ist nur Fallback.
