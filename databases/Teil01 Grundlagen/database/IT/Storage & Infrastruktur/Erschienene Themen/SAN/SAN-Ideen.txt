1. "LUN-Masking"
   - Ziel: Weise den Speicher dem richtigen Server zu.
   - Spielmechanik: 
     - Storage Array hat 100 TB.
     - LUN 1 (10 TB) erstellt.
     - Wer darf zugreifen? Server A.
     - Zugriff: Mapping WWN (World Wide Name) von Server A auf LUN 1.
     - Wenn Server B zugreift -> Datenkorruption! (Außer Cluster-Filesystem).
     - Messbarkeit: Exklusiver Zugriff gesichert.
   - Lernziel: Zugriffskontrolle im SAN.

2. "Fibre Channel vs iSCSI"
   - Ziel: Wähle die Technologie.
   - Spielmechanik: 
     - Budget: Klein, Ethernet vorhanden. -> iSCSI (IP-basiert).
     - Budget: Groß, Low Latency nötig. -> Fibre Channel (Lichtleiter, eigene Switche).
     - Messbarkeit: Passende Technologie-Wahl.
   - Lernziel: Protokolle im SAN.

3. "Der Zoning-Manager"
   - Ziel: Trenne die Geräte im FC-Switch (Sicherheit).
   - Spielmechanik: 
     - "Soft Zoning" (per WWN) oder "Hard Zoning" (per Port).
     - Ziel: Tape-Library darf nicht von Webserver gesehen werden.
     - Zone A: DB-Server + Storage.
     - Zone B: Mail-Server + Storage.
     - Messbarkeit: Geräte sehen nur ihre Partner.
   - Lernziel: Sicherheit im Fabric.

4. "Multipathing (MPIO)"
   - Ziel: Überlebe den Kabelbruch.
   - Spielmechanik: 
     - Server hat 2 HBA-Karten. Storage hat 2 Controller.
     - 4 Pfade möglich.
     - Ohne MPIO: Server sieht 4x die gleiche Festplatte (Verwirrung!).
     - Mit MPIO: Server sieht 1 Festplatte mit 4 Wegen.
     - Kabel ziehen -> Traffic schwenkt um.
     - Messbarkeit: Redundanter Zugriff.
   - Lernziel: Pfad-Redundanz.

5. "Block-Level-Access"
   - Ziel: Verstehe den Unterschied zum NAS.
   - Spielmechanik: 
     - NAS (File Level): Server sagt "Gib mir Datei host.txt". NAS liest und sendet.
     - SAN (Block Level): Server sagt "Gib mir Block 4096 bis 8192". Server baut Dateisystem selbst (NTFS/EXT4).
     - Szenario: Datenbank braucht Raw-Performance. -> SAN wählen.
     - Messbarkeit: Wahl der richtigen Speicherarchitektur.
   - Lernziel: Block vs File Storage.
