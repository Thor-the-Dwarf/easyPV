1. "Der menschliche Compiler"
   - Ziel: Spiele Computer im Kopf.
   - Spielmechanik: 
     - Code: `x=1; y=2; x=x+y; y=x-y;`
     - Tabelle ausfüllen:
     - Schritt 1: x=1, y=2.
     - Schritt 2: x=3.
     - Schritt 3: y=3-2=1.
     - Ergebnis: x=3, y=1.
     - Messbarkeit: Korrekte Werte-Tabelle.
   - Lernziel: Variablenverfolgung (Tracing).

2. "Off-by-One Fehlerfinder"
   - Ziel: Wie oft läuft die Schleife?
   - Spielmechanik: 
     - `FOR i = 0 TO 5`.
     - Läuft sie 5 mal oder 6 mal?
     - 0, 1, 2, 3, 4, 5. -> 6 mal!
     - Wenn Array Größe 5 hat -> Index Out of Bounds Fehler im letzten Schritt.
     - Messbarkeit: Korrekte Iterationsanzahl.
   - Lernziel: Grenzen testen.

3. "Pfad-Abdeckung (Branch Coverage)"
   - Ziel: Finde Input-Werte, um alle Zweige zu testen.
   - Spielmechanik: 
     - `IF (a > 10) THEN ... ELSE ...`
     - Testfall 1: a=5 (Else Zweig).
     - Testfall 2: a=11 (Then Zweig).
     - Testfall 3: a=10 (Grenzwert, Else Zweig).
     - Messbarkeit: 100% Code Coverage erreicht.
   - Lernziel: Testdaten-Erstellung.

4. "Die trockene Ausführung (Dry Run)"
   - Ziel: Finde den Logikfehler ohne PC.
   - Spielmechanik: 
     - Algorithmus zur Berechnung des Durchschnitts.
     - `Summe = 0`. `Zähler = 0`.
     - `Durchschnitt = Summe / Zähler`.
     - Fehler: Division durch Null beim Start!
     - Messbarkeit: Absturz im Kopf vorhersagen.
   - Lernziel: Edge Cases.

5. "Rekursions-Tiefe"
   - Ziel: Wann bricht es ab?
   - Spielmechanik: 
     - `Func(n): IF n=0 Return. ELSE Func(n-1)`.
     - Aufruf `Func(3)`.
     - Stack: 3 -> 2 -> 1 -> 0 (Abbruch).
     - Wie oft aufgerufen? 4 mal.
     - Messbarkeit: Korrekter Stack-Trace.
   - Lernziel: Rekursionsverständnis.
