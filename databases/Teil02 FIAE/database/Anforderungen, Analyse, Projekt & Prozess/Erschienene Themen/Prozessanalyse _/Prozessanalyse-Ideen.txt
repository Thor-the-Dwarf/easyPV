1. "IST-Zustand-Detektiv"
   - Ziel: Identifiziere Schwachstellen in einem aktuellen Geschäftsprozess.
   - Spielmechanik: 
     - "Daten werden dreimal manuell in Excel kopiert". -> Medienbruch / Redundanz.
     - "Dokument liegt 5 Tage beim Chef zur Unterschrift". -> Liegezeit / Flaschenhals.
     - Messbarkeit: Zuordnung der Schwachstellen-Art.
   - Lernziel: Prozessoptimierungspotenziale erkennen.

2. "Aktivitäten-Sortierer"
   - Ziel: Unterscheide wertschöpfende von nicht-wertschöpfenden Aktivitäten.
   - Spielmechanik: 
     - "Quellcode schreiben". -> Wertschöpfend.
     - "Fehlerkorrektur (Bugfix)". -> Nicht wertschöpfend (Nacharbeit).
     - "Warten auf Server-Bereitstellung". -> Verschwendung (Wartezeit).
     - Messbarkeit: Effizienz-Check des Prozesses.
   - Lernziel: Wertstromanalyse (Lean Principles).

3. "Durchlaufzeit-Rechner"
   - Ziel: Berechne die gesamte Durchlaufzeit eines Prozesses.
   - Spielmechanik: 
     - Schritt A (2h) -> Transport (1h) -> Schritt B (3h) -> Wartezeit (10h).
     - Frage: Wie lange dauert es bis zum Ende? -> 16h.
     - Messbarkeit: Korrekte Summierung aller Zeitanteile.
   - Lernziel: Verständnis von Durchlaufzeit, Bearbeitungszeit und Liegezeit.

4. "Prozess-Schnittstellen"
   - Ziel: Erkenne Input und Output von Prozessschritten.
   - Spielmechanik: 
     - Schritt: "Rechnung erstellen". -> Input: "Bestellungsdaten", Output: "Rechnungsbeleg".
     - Drag & Drop der richtigen Dokumente an Start und Ende.
     - Messbarkeit: Korrekte Datenfluss-Logik.
   - Lernziel: SIPOC-Modell (Supplier, Input, Process, Output, Customer).

5. "Rollen-Zuweiser"
   - Ziel: Ordne Rollen nach der RACI-Matrix zu.
   - Spielmechanik: 
     - Wer programmiert? -> Responsible.
     - Wer trägt die Gesamtverantwortung? -> Accountable.
     - Wer muss nur informiert werden? -> Informed.
     - Messbarkeit: Korrekte Anwendung der RACI-Kriterien.
   - Lernziel: Rollenverteilung in Prozessen.
