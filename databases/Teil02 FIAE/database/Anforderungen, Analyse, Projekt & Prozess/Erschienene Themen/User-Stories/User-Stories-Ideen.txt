1. "User Story-Baukasten"
   - Ziel: Baue eine User Story nach dem Standard-Schema.
   - Spielmechanik: 
     - Puzzle: "Als [Rolle]", "möchte ich [Ziel/Aktion]", "um [Nutzen/Grund] zu erreichen".
     - Bausteine kombinieren für verschiedene Szenarien.
     - Messbarkeit: Korrekte Syntax.
   - Lernziel: Struktur einer User Story.

2. "INVEST-Check"
   - Ziel: Prüfe die Qualität einer User Story.
   - Spielmechanik: 
     - Story: "Das System soll gut sein". -> Fail (nicht Testbar/Small).
     - Story: "Als Kunde möchte ich...". -> Pass (Independant, Negotiable...).
     - Messbarkeit: Erkennen von Verletzungen der INVEST-Kriterien.
   - Lernziel: Qualitätsstandards für Backlog Items.

3. "Akzeptanzkriterien-Fänger"
   - Ziel: Definiere Bedingungen für das "Ready" oder "Done".
   - Spielmechanik: 
     - Aufgabe: Schreibe 3 Kriterien für den "Warenkorb-Check".
     - Auswahl aus Pool: "Zahlung per Paypal möglich", "Gesamtsumme wird berechnet", "Farbe ist blau".
     - Messbarkeit: Relevanz der Kriterien für das Feature.
   - Lernziel: Definition of Done (DoD) / Akzeptanzkriterien.

4. "Story-Point-Poker"
   - Ziel: Schätze die Komplexität von User Stories.
   - Spielmechanik: 
     - Story A vs. Story B. 
     - Wähle Fibonacci-Zahlen (1, 2, 3, 5, 8, 13...).
     - Messbarkeit: Realistische Einschätzung im Vergleich zu Referenz-Stories.
   - Lernziel: Agiles Schätzen (Planning Poker).

5. "Persona-Match"
   - Ziel: Ordne User Stories den richtigen Personas zu.
   - Spielmechanik: 
     - Persona: "Oma Erna (technik-scheu)". vs. "Admin Alex (Power-User)".
     - Story: "Ein-Klick-Bestellung mit großen Buttons". -> Oma Erna.
     - Messbarkeit: Nutzerzentrierte Sichtweise.
   - Lernziel: Persona-Konzept in der Softwareentwicklung.
