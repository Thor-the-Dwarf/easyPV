1. "Rahmen-Puzzle"
   - Ziel: Kategorisiere Rahmenbedingungen (Zeit, Budget, Technik, Recht).
   - Spielmechanik: 
     - "Budget max. 50.000€". -> Ökonomisch.
     - "Nutzung von Azure Cloud". -> Technisch.
     - "Einhaltung der DSGVO". -> Rechtlich.
     - Messbarkeit: Korrekte Zuordnung in die Kategorien.
   - Lernziel: Verständnis von Projektgrenzen.

2. "Konflikt-Scanner"
   - Ziel: Finde Widersprüche zwischen Rahmenbedingungen und Zielen.
   - Spielmechanik: 
     - Ziel: "Mobile App". Rahmenbedingung: "Kein Budget für iOS-Devs".
     - Frage: Was ist das Problem? -> Ressourcen-Konflikt.
     - Messbarkeit: Erkennen von Planungsrisiken.
   - Lernziel: Machbarkeitsprüfung.

3. "Einfluss-Check"
   - Ziel: Erkenne, welche Rahmenbedingung welches Ziel einschränkt.
   - Spielmechanik: 
     - "Deadline in 2 Wochen" -> schränkt den [Umfang] ein.
     - "Vorgabe: Nur Open Source" -> schränkt die [Technologiewahl] ein.
   - Lernziel: Vernetztes Denken im Projektkontext.

4. "Budget-Tower"
   - Ziel: Verteile begrenzte Ressourcen auf Projektteile.
   - Spielmechanik: 
     - Du hast 10 "Münzen". 
     - UI Kostet 3, Backend 5, Testing 3. -> Budget überschritten!
     - Wo sparst du ein?
   - Lernziel: Ressourcenallokation unter Restriktionen.

5. "Technik-Check"
   - Ziel: Prüfe, ob die technische Infrastruktur zum Projekt passt.
   - Spielmechanik: 
     - Projekt: "KI-Modell lokal trainieren". Rahmenbedingung: "Standard-Laptops ohne GPU".
     - Entscheidung: "Möglich" oder "Nicht möglich (ohne Cloud)".
   - Lernziel: IT-Infrastruktur-Anforderungen.
