Anforderungen (UX-Game-Optimiert)
- Ziel: Spieler beherrschen die präzise Trennung von Zuständigkeiten (Model, View, Controller/ViewModel) in modernen Software-Architekturen.
- Game-Mechanic: "The Orchestra Conductor". Du bist der Dirigent eines Ensembles. Dir werden "Aktions-Partituren" (Aufgaben) zugespielt. Du musst sie sofort dem richtigen Musiker (Modul) zuweisen, bevor der Takt (Timer) endet.
- Challenge: 
  - Level 1 (Solo): Langsame Aufgaben wie "Hintergrundfarbe ändern" -> View.
  - Level 2 (Quintett): Aufgaben wie "Benutzerdaten validieren" oder "Berechnung starten". Wer schickt die Daten an wen?
  - Level 3 (Börsen-Sinfonie): Echtzeit-Datenströme (MVVM / Data Binding). Wenn du das Model direkt an die View bindest, ohne Transformation -> Penalty.

Arbeitspakete (UX-Game-Optimiert)
1. Interactive HUD und Scoring
- [ ] HUD-Elemente: "Harmonie-Faktor" (Accuracy), "BPM" (Speed-O-Meter), "Takt-Progress".
- [ ] Scoring: "Symphony-Bonus" für fehlerfreie Sequenzen. Abzug bei "Architektur-Dissonanz" (Zuständigkeits-Fail).

2. Screen-Layout (The Podium)
- [ ] `concert-hall -> conductor-desk -> role-zones -> task-stream`.
- [ ] Zones: Drei farblich pulsierende Bereiche (Model-Grün, View-Blau, Controller-Magenta).
- [ ] Task-Stream: Eintreffende Notenblätter mit Symbolik (Daten-DB, Auge-View, Zahnrad-Logic).

3. Kernvisuals und Effekte
- [ ] Premium-Klassik Ästhetik: Dunkles Holz, Samt-Rot, goldene Linien, Partikel-Effekte bei korrekter Zuweisung.
- [ ] "Conducting-Glow": Bei korrekter Wahl leuchtet der Weg der Daten hell auf.
- [ ] Distortion-Effekt: Bei falscher Wahl "verschiebt" sich das UI kurz schmerzhaft (visuelle Dissonanz).

4. Browser- und Device-Resilienz
- [ ] Mobile: "Swipe-to-Conduct" Geste.
- [ ] `min-height: 100dvh` Standard.

5. Content-Strategie (The Score-Pool)
- [ ] 20 reale Architektur-Szenarien von "Button Click" bis "Rest-API Call".

6. QA-Paket
- [ ] Test: Sind die Rollen-Zonen auf 6-Zoll Displays klar trennbar?
- [ ] Test: Fühlt sich das Timing bei Level 3 fordernd, aber nicht unfair an?

7. Abnahme-Kriterien
- [ ] Der Spieler verinnerlicht die Trennung von Logik und Präsentation.
- [ ] Das Interface wirkt wie ein exklusives High-Standard Game.
- [ ] Sofortiges Verständnis des Datenflusses durch visuelle Belohnung.
