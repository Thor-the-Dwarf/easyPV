1. "Pattern-Kategorisierer"
   - Ziel: Ordne Patterns ihren Typen zu (Erzeugung, Struktur, Verhalten).
   - Spielmechanik: 
     - Singleton -> Erzeugung.
     - Adapter -> Struktur.
     - Observer -> Verhalten.
     - Lernziel: Systematik der Gang of Four (GoF).

2. "Problem-Lösung-Match"
   - Ziel: Wähle das richtige Pattern für ein Problem.
   - Spielmechanik: 
     - Problem: "Nur eine Instanz der DB-Verbindung erlaubt". -> Singleton.
     - Problem: "Zwei inkompatible Interfaces verbinden". -> Adapter.
     - Lernziel: Patterns als Werkzeugkasten.

3. "Pattern-Vorteils-Quiz"
   - Ziel: Verstehe das "Wozu".
   - Spielmechanik: 
     - Code ohne Pattern vs. Code mit Pattern. 
     - Welcher ist leichter zu erweitern?
     - Lernziel: Nutzen von Patterns für die Wartbarkeit.

4. "Antipattern-Detektor"
   - Ziel: Erkenne schlechte Angewohnheiten.
   - Spielmechanik: 
     - "Spaghetti Code" oder "God Object" (eine Klasse macht alles).
     - Wie würde ein Pattern helfen?
     - Lernziel: Bewusstsein für Code-Qualität.

5. "Pattern-Syntax-Puzzle"
   - Ziel: Skizziere die Klassenstruktur eines Patterns.
   - Spielmechanik: 
     - Baue ein einfaches UML-Diagramm nach (z.B. Context/Strategy).
     - Lernziel: Strukturvisualisierung.
