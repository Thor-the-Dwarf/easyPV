Anforderungen (UX-Game-Optimiert)
- Ziel: Spieler beherrschen die Entkopplung von Objekterzeugung durch das Factory Pattern zur Umsetzung flexibler System-Builds.
- Game-Mechanic: "The Robot Assembler". Du steuerst eine automatisierte Fabrikhalle. Kunden bestellen "Export-Roboter". Du musst nicht wissen, wie die Roboter (PDF, CSV, JSON) innen aussehen – du musst nur den richtigen Auftrag in die Fabrik-Matrix schieben.
- Challenge: 
  - Level 1 (Production Line): Wähle den Typ. Die Fabrik spuckt das korrekte Modul aus.
  - Level 2 (Hidden Logic): Die Fabrik muss basierend auf versteckten Bedingungen (Dateigröße, Ziel-OS) selbst entscheiden, welche "Sub-Klasse" sie baut.
  - Level 3 (Expansion): Ein neues Produkt (XML) wird eingeführt. Beweise, dass dein Code (Open-Closed) nicht geändert werden muss, um das neue Produkt zu unterstützen.

Arbeitspakete (UX-Game-Optimiert)
1. Interactive HUD und Scoring
- [ ] HUD-Elemente: "Production-Rate", "Factory-Uptime", "Object-Integrity" (Check gegen Interface).
- [ ] Scoring: "Bulk-Order" Bonus für schnelle korrekte Zuweisungen. Abzug bei "Manual-Instancing" (wenn der Client versucht, Roboter selbst zu bauen).

2. Screen-Layout (The Factory Floor)
- [ ] `control-room -> factory-gate -> conveyor-belt -> shipping-bay`.
- [ ] Gate: Ein blickdichter Vorhang (The Interface), hinter dem die Magie (Factory) passiert.
- [ ] Conveyor: Hier fließen die "Bestellungen" (Requests) ein.

3. Kernvisuals und Effekte
- [ ] Industrial-High-Tech Ästhetik: Gebürstetes Metall, Warnstreifen, Kran-Animationen, zischende Dampf-Effekte.
- [ ] "Assemble-Flash": Wenn das Objekt hinter dem Gate fertiggestellt wird, blitzt ein starkes Arbeitslicht auf.
- [ ] Output-Ramp: Die fertigen Klassen-Vorschaubilder fahren auf Schienen aus der Fabrik.

4. Browser- und Device-Resilienz
- [ ] `CSS Masks` für den Fabrik-Vorhang Effekt.
- [ ] Mobile: "Swipe-to-Produce" Geste für maximale Haptik.

5. Content-Strategie (The Order-Log)
- [ ] 15 Produkttypen: Von "Windows-UI-Button" bis "Database-Connector-Pro".

6. QA-Paket
- [ ] Test: Ist die Trennung zwischen Client (Besteller) und Factory (Bauer) optisch durch die Wand klar genug?
- [ ] Test: Reagiert das UI flüssig auf schnelle Typ-Änderungen?

7. Abnahme-Kriterien
- [ ] Der Spieler versteht: Der Client kennt nur das Interface, die Fabrik die Details.
- [ ] Das Interface wirkt wie ein professioneller Industrie-Simulator.
- [ ] Hoher Lerneffekt für das Open-Closed Prinzip.
