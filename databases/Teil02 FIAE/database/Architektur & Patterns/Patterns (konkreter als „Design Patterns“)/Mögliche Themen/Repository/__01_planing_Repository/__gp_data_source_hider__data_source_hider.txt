Anforderungen (UX-Game-Optimiert)
- Ziel: Beherrschung des Repository-Patterns zur Kapselung von Persistenz-Details.
- Game-Mechanic: "The Library Ghost". Du bist in einer unendlichen Bibliothek. Suchanfragen der "User-Client" Geister flattern herein. Du musst die Daten holen, OHNE dass der Geist sieht, welcher Keller (Files, DB, API) gerade geöffnet wird.
- Challenge: 
  - Level 1: "Simple Fetch". Hol das Buch aus dem "Memory-Regal".
  - Level 2: "Backend-Swap". Die DB brennt! Schalte das Repository blitzschnell auf "Fallback-Cache" um, während die Anfragen weiterlaufen.
  - Level 3: "Complexity-Shield". Verstecke komplexe SQL-Joins hinter einer sauberen `getTopSellers()` Methode. Werden SQL-Secrets nach oben geleakt? (Penalty!).

Arbeitspakete (UX-Game-Optimiert)
1. Interactive HUD und Scoring
- [ ] HUD-Elemente: "Abstraction-Seal" (Integrity-Score), "Ghost-Happiness" (Satisfaction), "Backend-State".
- [ ] Scoring: +500 Pkt für "Leak-Free" Operations. Abzug bei "Leaking-Details" (wenn DB-Begriffe in der Domäne auftauchen).

2. Screen-Layout (The Secret Counter)
- [ ] `library-hall -> the-counter-interface -> the-hidden-storages -> data-packets`.
- [ ] Interface: Ein schmaler, sauberer Schlitz (Domain-Layer), durch den Anfragen kommen.
- [ ] Hidden-Storages: Verschiedene, düstere Bereiche im Hintergrund, die nur durch das Repository "beleuchtet" werden.

3. Kernvisuals und Effekte
- [ ] Dark-Academy / Mystery Ästhetik: Schwebendes Pergament, magische Siegel, Leuchteffekte bei Datenfluss.
- [ ] "Seal-Pulse": Ein goldenes Siegel auf dem Repository-Schlitz pulsiert, wenn die Kapselung perfekt ist.
- [ ] "Smoke-Effect": Bei Details-Leak (Fehler) verraucht die schöne Kachel-Optik in grauen Pixelmatsch.

4. Browser- und Device-Resilienz
- [ ] `backdrop-filter: blur()` für die Glass-Morphism Trennung der Layer.
- [ ] Mobile: Hochkant-Stacking (Client oben, Repo-Schild Mitte, Backends unten).

5. Content-Strategie (The Records)
- [ ] 12 Business-Objekte: `Customer`, `Invoice`, `StockItem`, `ProductRating`.

6. QA-Paket
- [ ] Test: Fühlt sich der Wechsel der Datenbanken "beiläufig" genug an?
- [ ] Test: Versteht der User den Zweck des "Schildes" (Abstraktion)?

7. Abnahme-Kriterien
- [ ] Der Spieler versteht die Persistenz-Ignoranz als strategischen Vorteil.
- [ ] Das Interface wirkt atmosphärisch, geheimnisvoll und hochwertig.
- [ ] Keine Verwirrung über die Trennung von DAO und Repository.
