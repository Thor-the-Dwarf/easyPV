Anforderungen (reverse engineered)
- Ziel: Implementierung einer Familie austauschbarer Algorithmen (Strategien), die zur Laufzeit gewechselt werden können, um Verzweigungs-Logik (If-Else) zu eliminieren.
- Kontext: Algorithmen-Austausch, Zahlungsarten, Sortier-Strategien, Verhaltensänderung.
- UX-Prinzip:
  - "The Weapon Wheel": In einem Spiel wählt man die Waffe (Strategie) für die aktuelle Situation. Die Hand (der Kontext), die sie hält, bleibt gleich.
  - "Slotting In": Der Spieler sieht einen offenen Slot im Hauptalgorithmus, in den verschiedene "Logik-Chips" (Strategien) passen.
  - "Zero-If Logic": Visuelle Demonstration, wie hunderte `if`-Abfragen durch eine einzige, saubere 호출 (Aufruf) an die aktuelle Strategie ersetzt werden.

Arbeitspakete (UX-Global-Optimiert)
1. Informationsarchitektur und Fokus-Hierarchie
- [ ] Sichtprioritaeten: 1 = Die aktive Strategie (The Gear), 2 = Das gemeinsame Interface (The Foundation), 3 = Der Ausführungskontext.
- [ ] Textbudget: Max. 5 Begriffe pro Strategie-Karte.
- [ ] "Decision HUD": Anzeige, welcher "Chip" gerade eingesteckt ist.

2. Screen-Layout (Container-Ebenen reduzieren)
- [ ] Struktur: `page -> strategy-wheel -> execution-arena -> logic-preview`.
- [ ] Wheel-Design: Ein kreisförmiges Menü in der Mitte (Desktop 50% Breite, Mobile Full-Screen Touch Zone).
- [ ] Arena: Hier wird das Ergebnis der gewählten Strategie live simuliert.

3. Browser- und Device-Resilienz
- [ ] `min-height: 100dvh` Standard.
- [ ] Mobile: "Circle-Swipe" Geste zum Durchschalten der Strategien.
- [ ] `touch-action: none` für verlustfreie Gestensteuerung im Rad.

4. Kernvisuals
- [ ] Gaming-Ästhetik: Neon-Menüs, taktische Overlays, Sound-Effekte beim Umschalten (shared_theme.css).
- [ ] Logik-Chips: Jede Strategie wird als physisches Modul mit verschiedenen Schaltkreis-Mustern dargestellt.
- [ ] High-Contrast "Active"-Glow.

5. Interaktion und Feedback
- [ ] Switch-Feedback: "Klick! Strategie wechselt auf 'Kreditkarte'. Die Gebührenberechnung passt sich sofort an."
- [ ] Aha-Moment: "Schau dir den Code an: Keine `if-else` Ketten mehr! Nur noch `payment.pay()`."
- [ ] Erfolg: "Maximale Flexibilität! Du kannst morgen eine 'Krypto-Strategie' hinzufügen, ohne den Checkout-Code zu berühren."

6. KPI-Klarheit
- [ ] Code-Cleanliness: "Conditional-Complexity: Minimal".
- [ ] Extensibility-Score: "Erweiterbarkeit: Unendlich".

7. Content-/Copy-Feinschliff
- [ ] Szenarien: "Zahlungsarten am Kiosk", "Routenplanung (Auto vs. Rad vs. Fuß)", "Rabatt-Aktionen".
- [ ] Tonalität: Taktisch, clever, modular.

8. QA-Paket
- [ ] Screenshot-Check: Ist das "Weapon Wheel" auf dem Smartphone mit dem Daumen gut bedienbar?
- [ ] Test: Reagiert die Simulations-Arena verzögerungsfrei auf den Strategiewechsel?

9. Abnahme-Kriterien
- [ ] Der Spieler begreift Polymorphie als mächtige Waffe gegen Unübersichtlichkeit.
- [ ] Das Interface wirkt wie ein exklusives Skill-Tree System aus einem Top-Spiel.
- [ ] Volle Klarheit über das Prinzip "Kapsle, was sich ändert".
