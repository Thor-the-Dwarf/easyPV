Anforderungen
1. "Server-vs-Client Match"
   - Ziel: Verständnis der komplementären Rollen von Client-seitiger (UX) und Server-seitiger (Security) Validierung zur Vermeidung von Umgehungs-Exploits.
   - Spielmechanik: 
     - Simulation einer Formular-Übermittlung.
     - Der Spieler sieht, wie Client-Checks einfach umgangen werden können (z.B. mittels DevTools).
     - Aufgabe: Implementierung robuster Validierungslogik auf dem Server (Validator-Middleware).
     - Filterung auf unerlaubte Datentypen und Längen.
     - Messbarkeit: Ablehnung von schädlichen Payloads trotz deaktiviertem JS im Browser.
   - Lernziel: Einhaltung des "Never Trust User Input" Prinzips in der Webentwicklung.
