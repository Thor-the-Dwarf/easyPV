1. "Server-vs-Client Match"
   - Ziel: Verstehe den Unterschied im Vertrauen.
   - Spielmechanik: 
     - Client-Check ist für UX. Server-Check ist für Sicherheit.
     - Umgehe den JS-Check (Mausklick) und schlage beim Server-Check fehl.
     - Lernziel: Pflicht zur serverseitigen Validierung.

2. "Whitelist-Guardian"
   - Ziel: Erlaube nur, was sicher ist.
   - Spielmechanik: 
     - Erlaube für das Feld "PLZ" nur 5 Ziffern. Alles andere wird abgelehnt.
     - Lernziel: Strenge Typ-Prüfung.

3. "Sanitize-the-Input"
   - Ziel: Bereinige gefährliche Zeichen.
   - Spielmechanik: 
     - Entferne `<script>` Tags aus einem Kommentar-Feld.
     - Lernziel: Schutz vor Script-Injektionen.

4. "Blacklist-Fail"
   - Ziel: Warum Blacklists oft scheitern.
   - Spielmechanik: 
     - Du hast "Script" verboten. Der Angreifer nutzt "sCrIpT".
     - Lernziel: Überlegenheit von Whitelists.

5. "Data-Length-Checker"
   - Ziel: Verhindere Buffer Overflows oder DB-Fehler.
   - Spielmechanik: 
     - Das Feld hat 50 Zeichen. Jemand schickt 10.000 MB. Blockiere das!
     - Lernziel: Ressourcen-Schonung und Stabilität.
