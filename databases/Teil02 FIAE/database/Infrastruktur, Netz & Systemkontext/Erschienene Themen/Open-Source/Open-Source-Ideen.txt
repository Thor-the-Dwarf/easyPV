1. "License-Selector-Match"

   - Spielmechanik: 
     - MIT (Viel Freiheit), GPL (Copyleft), Apache (Patentschutz).
     - Lernziel: Rechtliche Grundlagen von Software.

2. "Community-Contribution Flight"
   - Ziel: Wie funktioniert Open-Source Mitarbeit?
   - Spielmechanik: 
     - Forke ein Projekt -> Fixe einen Bug -> Erstelle einen Pull-Request.
     - Lernziel: Workflow in der Community.

3. "FOSS-vs-Proprietary"
   - Ziel: Kenne die Vor- und Nachteile.
   - Spielmechanik: 
     - Wer bietet Support? Wo kann man den Code ändern?
     - Lernziel: Wirtschaftliche Entscheidungen in der IT.

4. "Dependency-Scanner"
   - Ziel: Prüfe Lizenzen in deinem Projekt.
   - Spielmechanik: 
     - Finde heraus, ob eine Bibliothek in deinem Projekt GPL ist und deine Software "infiziert".
     - Lernziel: Lizenz-Compliance.

5. "Transparency-Check"
   - Ziel: Warum Open-Source oft sicherer ist.
   - Spielmechanik: 
     - "Vier-Augen-Prinzip": Tausende Entwickler haben den Code gesehen. Finde die Lücke.
     - Lernziel: Vertrauen durch Offenheit.
