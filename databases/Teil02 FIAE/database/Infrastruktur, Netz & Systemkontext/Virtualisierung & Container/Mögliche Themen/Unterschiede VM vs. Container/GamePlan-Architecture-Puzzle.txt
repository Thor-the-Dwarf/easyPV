Anforderungen (UX-Game-Optimiert)
- Ziel: Objektiver Vergleich von Virtualisierung (VM) und Containerisierung (Docker) hinsichtlich Performance, Isolation und Ressourcenverbrauch.
- Game-Mechanic: "The Architecture Sorter". Du bist der Logistik-Manager eines riesigen Frachthafens. Es kommen verschiedene Anfragen rein: "Ein komplettes, isoliertes OS für eine Bank" vs. "1000 kleine Microservices für einen Chat-Bot". Wähle das richtige Mittel: Packst du es in eine schwere, gepanzerte Kiste (VM mit eigenem Kernel) oder in einen leichten, stapelbaren Container (Docker, nutzt Host-Kernel)? Wer falsch wählt, verschwendet teuren Platz oder riskiert die Sicherheit.
- Challenge: 
  - Level 1 (The Speed Race): Starte 50 Instanzen einer App. Wer ist schneller? Vergleiche die Boot-Zeit von VM und Container.
  - Level 2 (The Isolation-War): Eine App ist instabil und versucht, den Kernel zu korrumpieren. Nutze die VM für maximale Isolation ("Hard-Isolation").
  - Level 3 (The Resource-Squeeze): Du hast nur noch 2 GB RAM frei. Packe so viele Instanzen wie möglich darauf. Optimiere den Pack-Faktor mit Containern ("Shared-Kernel-Bonus").

Arbeitspakete (UX-Game-Optimiert)
1. Interactive HUD und Scoring
- [ ] HUD-Elemente: "Spin-up-Timer", "RAM-Load (%)", "Security-Barrier-Strength", "Portability-Index".
- [ ] Scoring: "Efficiency-Expert" Bonus für die perfekte Wahl bei knappen Ressourcen. Abzug für "Over-Provisioning" (zu große VMs für kleine Tasks).

2. Screen-Layout (The Cargo Dock)
- [ ] `dock-crane -> container-stack -> vm-hangar -> resource-gauge`.
- [ ] Container-Stack: Bunte kleine Boxen, die sich hunderte Male stapeln lassen.
- [ ] VM-Hangar: Große, imposante Gebäude für die virtuellen Maschinen.

3. Kernvisuals und Effekte
- [ ] Industrial-Modern Ästhetik: Stahlträger, Container-Texturen, neon-blaue Scanlines, maritime Hafen-Sounds gemischt mit digitalen Klängen (Shared Theme kompatibel).
- [ ] "Pop-Spin-up": Container ploppen fast augenblicklich auf. VMs bauen sich langsam mit einem mechanischen Summen auf.
- [ ] "Boundary-Shield": VMs haben eine dicke, sichtbare Energieschutzwand. Container teilen sich ein gemeinsames Gitter (den Kernel-Boden).

4. Browser- und Device-Resilienz
- [ ] `CSS Flexbox` für das dynamische Stapeln der Container-Elemente.
- [ ] Mobile: "Drag-and-Drop" in den Hangar oder das Dock.

5. Content-Strategie (The Cargo-Manifests)
- [ ] 15 Architektur-Szenarien: Von "Legacy Windows App" bis "Cloud Native Microservice".

6. QA-Paket
- [ ] Test: Wird der Unterschied zwischen "Shared Kernel" und "Hardware-Abstraction" visuell greifbar?
- [ ] Test: Fühlt sich die Schnelligkeit der Container im Vergleich zu VMs "echt" an?

7. Abnahme-Kriterien
- [ ] Der Spieler begreift: Container sind für Geschwindigkeit & Dichte, VMs für Isolation & Kompatibilität.
- [ ] Das Interface wirkt wuchtig, rasant und technologiefokussiert.
- [ ] Fokus auf moderne Cloud-Infrastrukturen.
