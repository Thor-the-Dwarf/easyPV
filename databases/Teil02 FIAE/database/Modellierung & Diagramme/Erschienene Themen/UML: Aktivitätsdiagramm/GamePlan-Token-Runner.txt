Anforderungen (reverse engineered)
- Ziel: Praktische Validierung komplexer Prozesslogik durch Simulation eines Token-Laufs im UML-Aktivitätsdiagramm.
- Kontext: Prozess-Simulator, "Pathfinding" Spielmechanik.
- UX-Prinzip:
  - Das Diagramm (Graph) füllt den Visual-Viewport.
  - Der "Token" ist ein leuchtender Punkt, der dem Pfad folgt.
  - Entscheidungs-Stopp: An Decision-Nodes pausiert die Simulation für User-Input (Guard-Check).

Arbeitspakete (UX-Global-Optimiert)
1. Informationsarchitektur und Fokus-Hierarchie
- [ ] Sichtprioritaeten: 1 = Aktuelle Token-Position, 2 = Nächste mögliche Pfade, 3 = Eingabewerte (Guards).
- [ ] Textbudget: Max. 3 Wörter pro Aktion (z. B. "PDF erzeugen").
- [ ] Keine Anleitung; der "Start-Knoten" pulsiert zur Initialisierung.

2. Screen-Layout (Container-Ebenen reduzieren)
- [ ] Struktur: `page -> diagram-viewport -> guard-controls -> execution-log`.
- [ ] Viewport: Desktop 70% Breite, Mobile 60dvh Höhe. Zentriert auf den aktiven Knoten.
- [ ] Guard-Buttons: Großflächig (>=48px), erscheinen nur im Kontext einer Entscheidung.

3. Browser- und Device-Resilienz
- [ ] `min-height: 100dvh` Standard.
- [ ] SVG-Paths: Dynamische Dicke je nach Zoom-Level.
- [ ] Mobile: "Follow-Token" Modus (Viewport scrollt automatisch mit).

4. Kernvisuals
- [ ] Glow-Effekte: Aktive Kanten leuchten in der Farbe des Tokens.
- [ ] Animation: Smooth Transition zwischen den Knoten (Cubic-Bezier).
- [ ] Parallelitäts-Anzeige: Wenn der Token sich an einer Fork teilt, werden zwei Punkte sichtbar.

5. Interaktion und Feedback
- [ ] Sofort-Effekt: "Pfad gewählt: 'Saldo > 0' führt zum Buchungsschritt."
- [ ] Fehlermanagement: Wenn der falsche Pfad gewählt wird -> "Dead Lock" Animation und Tipp.
- [ ] Erfolg: Konfetti-Effekt beim Erreichen des "Final Nodes".

6. KPI-Klarheit
- [ ] Path-Efficiency: "Prozessdauer (simuliert): 45ms".
- [ ] Node-Coverage: "7 / 10 Aktionen durchlaufen".

7. Content-/Copy-Feinschliff
- [ ] IT-nahe Prozesse: "Login-Workflow", "CI/CD Pipeline", "Bestell-Storno".
- [ ] Tonalität: Logisch, prozessorientiert, dynamisch.

8. QA-Paket
- [ ] Screenshot-Check: Sind die Decision-Rhomben auf dem Smartphone klar von Aktionen (Rechtecke) unterscheidbar?
- [ ] Test: Läuft die Animation flüssig auf Low-End Devices?

9. Abnahmekriterien
- [ ] Die abstrakte Kontrollfluss-Logik wird durch die Token-Metapher "angreifbar".
- [ ] Das Interface wirkt wie ein State-of-the-Art Prozess-Monitor.
- [ ] Maximale Visualisierungs-Qualität.
