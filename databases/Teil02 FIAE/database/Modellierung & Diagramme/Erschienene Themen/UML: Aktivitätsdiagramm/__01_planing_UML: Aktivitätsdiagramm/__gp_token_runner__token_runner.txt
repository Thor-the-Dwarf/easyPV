Anforderungen (UX-Game-Optimiert)
- Ziel: Praktisches Verständnis von UML-Aktivitätsdiagrammen durch Simulation komplexer Kontrollflüsse und Forks/Joins.
- Game-Mechanic: "The Pipeline Runner". Du steuerst ein Datenpaket (Der Token) durch eine verzweigte Pipeline. An Decision-Nodes (Guards) musst du die richtigen Bedingungen erfüllen, an Forks teilt sich dein Token – beide müssen ans Ziel kommen!
- Challenge: 
  - Level 1 (Linear): Ein einfacher Workflow ohne Abzweigungen zum Aufwärmen.
  - Level 2 (Decision-Maze): Triff Guards wie `IsAuthorized` oder `StockAvailable`. Wählst du falsch, landet der Token im "Dead-End" und das Level resettet.
  - Level 3 (Parallel-Race): Ein Fork teilt den Token. Du musst beide gleichzeitig managen (Sync-Check), bevor sie am Join-Knoten wieder vereint werden.

Arbeitspakete (UX-Game-Optimiert)
1. Interactive HUD und Scoring
- [ ] HUD-Elemente: "Flow-Pressure" (Timer), "Token-Health" (Accuracy), "Sync-Status" (für Parallelismus).
- [ ] Scoring: "Fast-Flow" Bonus für schnelle Guard-Entscheidungen. Punkte für jede korrekt durchlaufene Aktion.

2. Screen-Layout (The Pipeline Map)
- [ ] `void-space -> diagram-canvas -> guard-selector -> execution-log`.
- [ ] Canvas: Ein technisches Blaupausen-Gitter, auf dem das Diagramm als leuchtende Röhrenstruktur liegt.
- [ ] Selector: Erscheint bei Decision-Nodes als große Overlay-Buttons ("TRUE" / "FALSE").

3. Kernvisuals und Effekte
- [ ] Cyber-Automation Ästhetik: Dunkler Hintergrund, neon-grüne Linien für aktive Pfade, rote Warnungen bei Dead-Locks.
- [ ] "Token-Glow": Der Token fließt als Lichtpuls durch die Röhren. Bei einem Join verschmelzen zwei Pulse zu einem größeren.
- [ ] "Join-Activation": Der Join-Knoten pulsiert gelb, während er auf den zweiten Token wartet.

4. Browser- und Device-Resilienz
- [ ] SVG-basierte Grafik für verlustfreies Zoomen (Desktop Center, Mobile Follow-Mode).
- [ ] `touch-action: pan-x pan-y` für flüssiges Verschieben des Diagramms.

5. Content-Strategie (The Workflows)
- [ ] 12 Szenarien: "CI/CD Pipeline", "User-Registration", "E-Mail Dispatcher", "Börsen-Auftrag".

6. QA-Paket
- [ ] Test: Ist das Timing zwischen Fork und Join fair (bleibt genug Zeit für beide Pfade)?
- [ ] Test: Sind die Tooltips für Guards auf kleinen Screens gut lesbar?

7. Abnahme-Kriterien
- [ ] Der Spieler versteht UML-Prozesslogik als dynamischen Fluss.
- [ ] Das Interface wirkt wie ein moderner Prozess-Debugger.
- [ ] Hohe Lerneffizienz für Parallelität und Bedingungen.
