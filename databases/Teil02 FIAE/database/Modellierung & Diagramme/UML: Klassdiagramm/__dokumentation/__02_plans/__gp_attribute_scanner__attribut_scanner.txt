Anforderungen (UX-Game-Optimiert)
- Ziel: Präzise Analyse von OOP-Code und Übersetzung in formale UML-Klassendiagramm-Syntax (Sichtbarkeit, Datentypen, Multiplizitäten).
- Game-Mechanic: "The Blueprint Auditor". Du bist der Qualitätsprüfer für Software-Blaupausen. Code-Snippets (Java/C#) fließen über den Schirm. Du musst sie scannen und die UML-Klassenkarte mit den korrekten Symbolen (+, -, #, ~) und Multiplizitäten (`0..*`) "stempeln".
- Challenge: 
  - Level 1 (Public-Private): Einfache Felder extrahieren. Markiere Sichtbarkeit und Datentyp.
  - Level 2 (The Multiplicator): Handle Listen und Arrays. Setze die Multiplizität korrekt (z. B. `List<Order>` -> `[0..*]`).
  - Level 3 (Inheritance & Nesting): Analyse geschützte (#) Attribute in Unterklassen und komplexe verschachtelte Typen.

Arbeitspakete (UX-Game-Optimiert)
1. Interactive HUD und Scoring
- [ ] HUD-Elemente: "Conformity-Check" (Accuracy), "Scan-Speed" (Time), "UML-Compliance" (XP).
- [ ] Scoring: "Double-Stamp" Bonus für fehlerfreie Extraktion ganzer Klassen. Abzug bei Syntax-Fehlern (z. B. falsche Doppelpunkt-Position).

2. Screen-Layout (The Auditor's Desk)
- [ ] `blueprint-viewer -> source-code-terminal -> uml-class-card -> syntax-stamp-tool`.
- [ ] Code-Terminal: Monospace-Font mit farbigem Syntax-Highlighting.
- [ ] Class-Card: Ein physisch wirkendes Dokument, das sich füllt.

3. Kernvisuals und Effekte
- [ ] Blueprint-Blue Ästhetik: Dunkle Blaupausen-Textur, weiße Tusche-Linien, Raster-Hintergrund.
- [ ] "Scan-Beam": Ein Laserstrahl fährt über den Code, wenn ein Attribut erkannt wurde.
- [ ] "Stamp-Effect": Ein physisches Stampfen und Geräusch, wenn ein Symbol (+/-) auf die Karte gesetzt wird.

4. Browser- und Device-Resilienz
- [ ] `font-family: 'Roboto Mono', monospace` für den Code-Kontext.
- [ ] Mobile: "Tap-to-Label" Mechanik. Code oben fixiert (Split-Screen).

5. Content-Strategie (The Syntax-Cases)
- [ ] 20 Snippets: Von `private int age;` bis `protected Set<Category> tags;`.

6. QA-Paket
- [ ] Test: Sind die kleinen UML-Symbole auf Smartphone-Screens ohne zu zoomen erkennbar?
- [ ] Test: Fühlt sich das "Übertragen" von Code zu UML wie eine Belohnung an?

7. Abnahme-Kriterien
- [ ] Der Spieler verankert die UML-Notation (Sichtbarkeit : Name : Typ) im Langzeitgedächtnis.
- [ ] Das Interface wirkt wie ein professionelles Architektur-Werkzeug.
- [ ] Fehlerfreies Mapping von OOP auf UML.
