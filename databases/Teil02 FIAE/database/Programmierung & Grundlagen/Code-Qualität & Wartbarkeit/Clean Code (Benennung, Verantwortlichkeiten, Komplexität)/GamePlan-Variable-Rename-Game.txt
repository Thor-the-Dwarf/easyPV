Anforderungen (UX-Game-Optimiert)
- Ziel: Praktische Anwendung aussagekräftiger Bezeichner nach Clean-Code-Prinzipien zur Steigerung der Wartbarkeit.
- Game-Mechanic: "The Name Auditor". Du bist der Prüfer in einem chaotischen Software-Archiv. Der Code funktioniert, aber die Variablen heißen `x`, `data1`, `tmp_f`. Du musst die "wahre Identität" dieser Variablen aus dem Kontext (Kommentare, Nutzung) ermitteln und ihnen treffende Business-Namen geben.
- Challenge: 
  - Level 1 (The Generic Fog): Benenne `int d; // days since login` in `daysSinceLastLogin` um.
  - Level 2 (The Bad Prefix): Entferne ungarische Notation oder unnötige Präfixe/Suffixe (`strName`, `ListUser`).
  - Level 3 (The Domain Master): Finde präzise Namen für komplexe Datenstrukturen in einem fiktiven Versicherungs-Szenario. Ein falscher Name führt zu "Semantic-Errors" (Penalty).

Arbeitspakete (UX-Game-Optimiert)
1. Interactive HUD und Scoring
- [ ] HUD-Elemente: "Readability-Rank" (Flesch-Index für Code), "Audit-Progress", "Naming-Consistency".
- [ ] Scoring: "Clarity-Bonus" für Namen, die ohne Kommentar sofort verständlich sind. Punktabzug bei zu langen oder zu kurzen Namen.

2. Screen-Layout (The Archive)
- [ ] `scroll-vault -> code-scanner -> name-input-field -> audit-log`.
- [ ] Code-Scanner: Ein Ausschnitt von 10-15 Zeilen mit hervorgehobenen "Bad Names" (rot blinkend).
- [ ] Name-Input: Ein Feld, das bei jedem Tippen eine "Clarity-Vorschau" (Was assoziiert eine KI damit?) zeigt.

3. Kernvisuals und Effekte
- [ ] Retro-Detective Ästhetik: Schreibmaschinen-Schriftart, Aktenordner-Icons, Kaffeetassen-Flecken (shared_theme.css), gedimmtes warmes Licht.
- [ ] "Scan-Effect": Eine virtuelle Lupe fährt über den Code und lässt schlechte Namen "heraustreten".
- [ ] "Ink-Correction": Wenn ein Name korrigiert wird, schreibt eine virtuelle Feder den neuen Namen über den alten.

4. Browser- und Device-Resilienz
- [ ] `spellcheck="false"` für das Input-Feld, um Browser-Unterstreichungen zu unterdrücken.
- [ ] Mobile: "Tap-to-Edit" Mechanik für die Variablennamen.

5. Content-Strategie (The Records)
- [ ] 20 reale Code-Szenarien: Von "E-Commerce Warenkorb" bis "IoT-Sensor-Stream".

6. QA-Paket
- [ ] Test: Ist die visuelle Bestätigung ("Reading-Ease increases!") motivierend genug für den Spieler?
- [ ] Test: Sind die Eingabefelder auf Mobile groß genug für ermüdungsfreies Tippen?

7. Abnahme-Kriterien
- [ ] Der Spieler verankert: "Namen sind die halbe Dokumentation".
- [ ] Das Interface wirkt nostalgisch-wertvoll und beruhigend.
- [ ] Hoher Lerneffekt für professionelle Team-Programmierung.
