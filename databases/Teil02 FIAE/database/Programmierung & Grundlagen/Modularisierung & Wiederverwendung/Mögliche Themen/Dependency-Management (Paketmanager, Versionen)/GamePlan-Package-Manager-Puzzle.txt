Anforderungen (UX-Game-Optimiert)
- Ziel: Spieler beherrschen das Dependency-Management, Versions-Constraints (SemVer) und das Auflösen von Konflikten.
- Game-Mechanic: "The Dependency Maze". In deinem Projekt-Labor wuchert ein Netz aus Ranken (Abhängigkeiten). Jede Ranke hat eine Version (z. B. `^1.2.0`). Wenn zwei Bibliotheken die gleiche Ranke in inkompatiblen Versionen wollen, gibt es einen "Version-Clash". Du musst die Ranken so stutzen (entfernen) oder flicken (aktualisieren), dass das Projekt stabil bleibt.
- Challenge: 
  - Level 1 (The Fresh Install): Installiere die korrekten Pakete für ein "Shopping-Cart" Modul.
  - Level 2 (The Security-Hole): Ein Paket hat eine `Critical Vulnerability`. Update es auf die nächste sichere Version, ohne andere Pakete zu brechen.
  - Level 3 (The Diamond Dependency): Finde eine Version, die zwei konkurrierende Anforderungen (z. B. `~2.1.0` vs. `>2.0.5`) gleichzeitig erfüllt.

Arbeitspakete (UX-Game-Optimiert)
1. Interactive HUD und Scoring
- [ ] HUD-Elemente: "Build-Stability" (%), "Vulnerability-Alerts", "Package-Count (Weight)".
- [ ] Scoring: "Lean-Dependency" Bonus für Projekte mit minimalem Overhead. Abzug bei "Dependency-Hell" (unauflösbare Konflikte).

2. Screen-Layout (The Laboratory)
- [ ] `growth-chamber -> dependency-tree-graph -> tool-bench (CLI-Terminal) -> lockfile-safe`.
- [ ] Dependency-Tree: Ein interaktiver Graph mit Knoten (Paketen) und Linien (Abhängigkeiten). Rote Linien markieren Konflikte.
- [ ] Tool-Bench: Hier wählst du Befehle wie `npm install`, `audit fix` oder `update`.

3. Kernvisuals und Effekte
- [ ] Bio-Tech Ästhetik: Organische Formen (Ranken), leuchtende Zellen, Glas-Reagenzgläser, technische Anzeigen.
- [ ] "Conflict-Spark": Wenn zwei Versionen kollidieren, springen zwischen den Knoten kleine rote Blitze.
- [ ] "Growth-Effect": Wenn ein Paket erfolgreich installiert wird, wächst eine neue, grüne Ranke in den Graph.

4. Browser- und Device-Resilienz
- [ ] `canvas` oder `SVG` für das flüssige Zeichnen des Abhängigkeits-Graphen.
- [ ] Mobile: "Pinch-to-Zoom" im Graph.

5. Content-Strategie (The Scenarios)
- [ ] 15 Fälle aus dem echten Leben (npm, Maven, Pip-Szenarien).

6. QA-Paket
- [ ] Test: Ist die Bedeutung von `^` vs. `~` vs. `*` im Spiel intuitiv erfassbar?
- [ ] Test: Reagiert der Graph verzögerungsfrei auf Änderungen im "Lockfile"?

7. Abnahme-Kriterien
- [ ] Der Spieler begreift SemVer und das Risiko externer Abhängigkeiten.
- [ ] Das Interface wirkt futuristisch, "lebendig" und hochspezialisiert.
- [ ] Volle Punkte durch strategisches Versions-Management.
