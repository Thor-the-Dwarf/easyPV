Anforderungen (UX-Game-Optimiert)
- Ziel: Spieler beherrschen den Unterschied zwischen synchroner (blockierender) und asynchroner (nicht-blockierender) Programmierung zur Vermeidung von UI-Freezes.
- Game-Mechanic: "The Traffic Control". Du steuerst die Hauptstraße (Der Main-Thread). Schwere LKW (Blocking-Requests) bremsen den Verkehr aus. Du musst sie auf die "Nebenstraßen" (Worker-Threads) umleiten, damit die kleinen PKW (UI-Interaktionen) flüssig weiterfahren können.
- Challenge: 
  - Level 1 (Single-Lane): Ein langer Request blockiert alles. Schalte manuell auf `Async` um, um die Blockade zu lösen.
  - Level 2 (The Rush Hour): 10 Requests gleichzeitig. Wer kommt in den Pool, wer bleibt blockierend? Achte auf die "Freeze-Anzeige".
  - Level 3 (Callback-Hell): Verschachtelte asynchrone Rufe. Ordne die Callbacks/Promises so, dass das Ergebnis am Ende wieder auf die Hauptstraße (Main-Thread Update) fließt.

Arbeitspakete (UX-Game-Optimiert)
1. Interactive HUD und Scoring
- [ ] HUD-Elemente: "System-Responsiveness" (60 FPS Gauge), "Worker-Load", "Async-Queue-Length".
- [ ] Scoring: "Smooth-Flow" Bonus für 100% UI-Reaktionsfähigkeit. Punktabzug bei jeder Sekunde "Lag" (Spinning Wheel).

2. Screen-Layout (The Digital Highway)
- [ ] `skyline -> main-thread-highway -> worker-service-road -> task-dashboard`.
- [ ] Highway: Eine animierte Straße, auf der User-Interaktionen als bunte Pixel-Autos fließen.
- [ ] Dashboard: Hier erscheinen die "Befehle" (Code-Snippets), die sortiert werden müssen.

3. Kernvisuals und Effekte
- [ ] Neon-City Ästhetik: Dunkler Asphalt, glühende Lichter, künstliche Horizon-Linien, Synthwave-Vibes (shared_theme.css).
- [ ] "Freeze-Alarm": Bei blockierendem Code friert das Bild ein und bekommt einen frostigen Blaufilter-Effekt.
- [ ] "Async-Launch": Ein Klick auf "Async" lässt einen LKW mit einem Raketenboost auf die Nebenstraße schießen.

4. Browser- und Device-Resilienz
- [ ] `requestAnimationFrame` für die flüssige Darstellung der 60 FPS Gauge.
- [ ] Mobile: "Tap-to-Reroute" Mechanik.

5. Content-Strategie (The Tasks)
- [ ] 15 Code-Snippets: Von `fs.readFileSync` (Blocking) bis `fetch().then()` (Async).

6. QA-Paket
- [ ] Test: Fühlt sich der "UI-Freeze" im Spiel nervig genug an, um den Lerneffekt zu verstärken?
- [ ] Test: Sind die Task-Karten auf dem Handy gut sortierbar?

7. Abnahme-Kriterien
- [ ] Der Spieler versteht: Blocking-Code auf dem Main-Thread ist der Feind.
- [ ] Das Interface wirkt wie ein stylischer Arcade-Titel.
- [ ] Maximale Punkte durch "Interference-Free" Programmierung.
