Anforderungen (UX-Game-Optimiert)
- Ziel: Messung und Reduktion der zyklomatischen Komplexität (McCabe) zur Steigerung der Testbarkeit und Wartbarkeit.
- Game-Mechanic: "The Complexity Clock". Du bist der Mechaniker in einer Dampfmaschinen-Halle. Jede Weiche (`IF`), jeder Kolben (`LOOP`) und jeder Ausgang (`RETURN`) erhöht den Druck (Die Komplexität). Wenn die Maschine zu viele Wege hat, explodiert die "Komplexitäts-Uhr". Du musst den Code so umbauen, dass der Druck (Die Zahl) im grünen Bereich bleibt.
- Challenge: 
  - Level 1 (The Simple Pressure): Zähle die Pfade durch eine kleine Funktion. Liegt sie unter 10?
  - Level 2 (The Nested Hub): Hilfe! Eine Funktion hat 15 verschachtelte `IF`s. Zerschneide sie in mehrere, einfache Maschinen (Methodenextraktion).
  - Level 3 (The Logic-Labyrinth): Ein Pfad mit 25 Verzweigungen. Finde die logischen Abkürzungen (Early Returns), um die Uhr wieder auf "Safe" zu stellen.

Arbeitspakete (UX-Game-Optimiert)
1. Interactive HUD und Scoring
- [ ] HUD-Elemente: "McCabe-Gauge (Pressure)", "Path-Complexity-Display", "Maintenance-Risk Warnleuchte".
- [ ] Scoring: "Simplifier-Bonus" für Funktionen mit Komplexität < 5. Abzug bei "Spaghetti-Score" (zu hohe Verschachtelung).

2. Screen-Layout (The Engine Room)
- [ ] `steam-pipes (Control Flow Graph) -> logic-gauges -> code-editor -> pressure-release-valve`.
- [ ] Steam-Pipes: Eine Visualisierung des Kontrollflusses als Rohrsystem.
- [ ] Logic-Gauges: Analoge Anzeigen, die beim Tippen zittern und steigen/fallen.

3. Kernvisuals und Effekte
- [ ] Steampunk-Industrial Ästhetik: Messing, Dampf, zitternde Nadeln auf Anzeigen, mechanisches Klappern, zischende Geräusche.
- [ ] "Pressure-Spike": Wenn du eine `ELSE`-Abzweigung hinzufügst, schlägt die Nadel hart nach rechts aus und Dampf entweicht.
- [ ] "Safe-Vibration": Ein sanftes Brummen, wenn der Code im optimalen Komplexitätsbereich liegt.

4. Browser- und Device-Resilienz
- [ ] `CSS Shaders` (falls möglich) oder Grainy-Filter für den Vintage-Look (Shared Theme kompatibel).
- [ ] Mobile: "Swipe-to-Refactor" Geste.

5. Content-Strategie (The Steam-Logs)
- [ ] 15 reale Komplexitätsfälle: Von "Finanz-Berechnung" bis "Game-Enemy-KI".

6. QA-Paket
- [ ] Test: Ist die visuelle Verbindung zwischen "Code-Änderung" und "Nadel-Ausschlag" verzögerungsfrei?
- [ ] Test: Hilft der Kontrollfluss-Graph (Rohre) beim Verständnis der McCabe-Zahl?

7. Abnahme-Kriterien
- [ ] Der Spieler begreift: Einfacher Code ist besserer Code.
- [ ] Das Interface wirkt atmosphärisch, wuchtig und technisch faszinierend.
- [ ] Fokus auf saubere Code-Strukturierung.
