Anforderungen (UX-Game-Optimiert)
- Ziel: Spieler beherrschen die Methode des Code-Reviews zur Qualitätssteigerung und Fehlerprävention (Vier-Augen-Prinzip).
- Game-Mechanic: "The Four-Eyes-Scan". Du bist der "Code-Prophet". Ein Kollege reicht seinen Code ein. Du schaust durch eine magische Brille (Die Review-View), die "Gerüche" (Smells), "Unsauberkeiten" und "Logik-Löcher" farblich markiert. Du musst entscheiden: "LGTM" (Looks Good To Me) oder "Request Changes"?
- Challenge: 
  - Level 1 (The Typo-Fix): Finde einfache Flüchtigkeitsfehler und vergessene Kommentare.
  - Level 2 (The Logic-Gap): Der Code sieht gut aus, aber eine `NullPointer`-Gefahr lauert im Schatten. Markiere sie!
  - Level 3 (The Performance-Killer): Finde die `O(n²)` Schleife im Herzstück. Schlage eine bessere Lösung vor, um den "Propheten-Rang" zu steigen.

Arbeitspakete (UX-Game-Optimiert)
1. Interactive HUD und Scoring
- [ ] HUD-Elemente: "Insight-Level", "Review-Thoroughness", "Approval-Rating".
- [ ] Scoring: "Eagle-Eye" Bonus für das Finden von Fehlern, die automatisierte Tests übersehen haben. Abzug bei "Nitpicking" (unwichtige Stilfragen überbewerten).

2. Screen-Layout (The Zen Garden)
- [ ] `meditation-mat -> dual-code-pane -> comment-scrolls -> floating-action-icons`.
- [ ] Dual-Code-Pane: Links der neue Code, rechts die Kommentare (Desktop 50/50, Mobile Tab-Switch).
- [ ] Floating-Action-Icons: "Approve", "Comment", "Reject" schweben als gläserne Orbs am Rand.

3. Kernvisuals und Effekte
- [ ] Zen-Tech Ästhetik: Minimalismus, sanfte Pastelltöne, Wasser-Animationen im Hintergrund, ruhiger Ambient-Sound.
- [ ] "Glass-Overlay": Wenn du auf eine Codezeile klickst, öffnet sich ein transparentes Fenster für deinen Kommentar.
- [ ] "Enlightenment": Nach einem perfekten Review erstrahlt der Code in einem sanften weißen Licht.

4. Browser- und Device-Resilienz
- [ ] `backdrop-filter: blur()` für die gläsernen UI-Elemente.
- [ ] Mobile: "Swipe-to-Comment" Funktion.

5. Content-Strategie (The Manuscripts)
- [ ] 20 Code-Auszüge mit versteckten "Sünden": Von SQL-Injection bis Hardcoded Secrets.

6. QA-Paket
- [ ] Test: Fühlt sich die Interaktion "wertschätzend" an (positives Feedback für den Reviewer)?
- [ ] Test: Sind die Code-Diffs (Highlighting) auf Mobile noch präzise anklickbar?

7. Abnahme-Kriterien
- [ ] Der Spieler begreift: Reviews sind Training für das eigene Auge.
- [ ] Das Interface wirkt modern, beruhigend und hochgradig ästhetisch.
- [ ] Fokus auf konstruktive Softwarequalität.
