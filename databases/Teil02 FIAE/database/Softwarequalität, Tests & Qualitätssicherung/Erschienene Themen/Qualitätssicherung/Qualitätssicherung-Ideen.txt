1. "QS-Massnahmen-Match"
   - Ziel: Unterscheide konstruktive und analytische QS-Maßnahmen.
   - Spielmechanik: 
     - "Einsatz von Programmierrichtlinien". -> Konstruktiv.
     - "Durchführung von Unit-Tests". -> Analytisch.
     - Lernziel: Methoden der Qualitätssicherung.

2. "Review-Prozess-Runner"
   - Ziel: Durchlaufe einen strukturierten Review.
   - Spielmechanik: 
     - Autoren, Gutachter, Moderator. Wer macht was?
     - Lernziel: Rollenverteilung in der QS.

3. "Bug-lifecycle-Puzzle"
   - Ziel: Verwalte den Status eines Fehlers.
   - Spielmechanik: 
     - New -> Assigned -> Fixed -> Verified -> Closed.
     - Lernziel: Prozess der Fehlerbehebung.

4. "Kosten-der-Fehler-Kurve"
   - Ziel: Verstehe, warum frühes Finden von Fehlern günstiger ist.
   - Spielmechanik: 
     - Klicke auf den Zeitpunkt, an dem ein Fehler gefunden wird, und sieh die fiktiven Kosten.
     - Lernziel: Wirtschaftlichkeit der QS.

5. "Checklisten-Master"
   - Ziel: Nutze Vorlagen für konsistente Qualität.
   - Spielmechanik: 
     - Hake alle Kriterien ab, bevor ein Release freigegeben wird.
     - Lernziel: Standardisierte Prüfprozesse.
