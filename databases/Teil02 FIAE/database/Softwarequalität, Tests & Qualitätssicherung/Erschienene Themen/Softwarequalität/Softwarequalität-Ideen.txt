1. "ISO-25010-Match"
   - Ziel: Lerne die Qualitätsmerkmale nach ISO 25010 (Funktionalität, Zuverlässigkeit, Benutzbarkeit, etc.).
   - Spielmechanik: 
     - "Wie gut lässt sich die Software an neue Anforderungen anpassen?". -> Match: Änderbarkeit.
     - "Fällt das System oft aus?". -> Match: Zuverlässigkeit.
     - Lernziel: Standardisierte Qualitätskriterien.

2. "Performance-Checker"
   - Ziel: Verstehe die Effizienz von Software.
   - Spielmechanik: 
     - Zwei Algorithmen lösen das gleiche Problem. Welcher verbraucht weniger RAM/CPU?
     - Lernziel: Ressourcen-Effizienz.

3. "Usability-Fahnder"
   - Ziel: Identifiziere Hürden für den Endbenutzer.
   - Spielmechanik: 
     - Finde den Fehler im UI-Design (z.B. kein Kontrast, verwirrende Labels).
     - Lernziel: Benutzbarkeit als Qualitätsmerkmal.

4. "Maintainability-Quiz"
   - Ziel: Erkenne wartbaren Code.
   - Spielmechanik: 
     - Vergleiche zwei Code-Snippets. Welches ist leichter zu lesen und zu erweitern?
     - Lernziel: Änderbarkeit & Wartbarkeit.

5. "Portability-Challenge"
   - Ziel: Software auf verschiedenen Plattformen lauffähig machen.
   - Spielmechanik: 
     - Läuft die App auch auf mobilen Geräten oder anderen Browsern?
     - Lernziel: Übertragbarkeit.
