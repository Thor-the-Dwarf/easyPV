Anforderungen (UX-Game-Optimiert)
- Ziel: Systematisches Entwerfen von Testfällen mit Vorbedingungen, Testdaten, Handlungsschritten und erwarteten Ergebnissen.
- Game-Mechanic: "The Scenario Writer". Du bist der Drehbuchautor für einen Stunt-Film. Dein Stuntman (Die Software) muss gefährliche Manöver ausführen. Du schreibst das Drehbuch (Den Testfall): "Wenn das Auto mit 100km/h gegen die Wand fährt (Input), muss der Airbag aufgehen (Expected Result)." Wenn dein Drehbuch ungenau ist, passiert ein Unfall!
- Challenge: 
  - Level 1 (The Standard Stunt): Schreibe einen Testfall für einen erfolgreichen Login (Positive Test).
  - Level 2 (The Crash-Test): Schreibe einen Testfall für eine ungültige Kreditkartennummer (Negative Test). Berücksichtige die Fehlermeldung.
  - Level 3 (The Epic Sequence): Ein komplexer Multi-Step-Stunt. "Warenkorb füllen -> Rabattcode nutzen -> Versandart wählen". Jeder Schritt braucht eine exakte Beschreibung.

Arbeitspakete (UX-Game-Optimiert)
1. Interactive HUD und Scoring
- [ ] HUD-Elemente: "Story-Clarity", "Safety-Score", "Precondition-Alert".
- [ ] Scoring: "Director's-Cut" Bonus für vollständige und präzise Testbeschreibungen. Abzug bei "Vague-Steps" (zu ungenaue Anweisungen).

2. Screen-Layout (The Movie Set)
- [ ] `storyboard-view -> script-editor (Test Case Template) -> stunt-preview (Expected Result Animation)`.
- [ ] Script-Editor: Felder für ID, Titel, Vorbedingung, Schritte, Erwartetes Ergebnis.
- [ ] Stunt-Preview: Eine kleine Animation, die zeigt, was bei Erfolg/Fehler passieren sollte.

3. Kernvisuals und Effekte
- [ ] Cinema-Vintage Ästhetik: Regiestuhl-Icon, Filmklappe, Spotlights, Schwarz-Weiß-Look mit Farbakzenten (shared_theme.css kompatibel).
- [ ] "Action-Impact": Wenn ein Testfall fertig ist, schlägt die Filmklappe zu ("Action!").
- [ ] "Script-Check": Korrekte Felder leuchten golden auf, fehlende Informationen blinken rot wie eine Warnleuchte am Set.

4. Browser- und Device-Resilienz
- [ ] `CSS Flexbox` für das Formular-Design (Responsives Grid).
- [ ] Mobile: "Voice-to-Text" Option (simuliert) für die schnelle Eingabe.

5. Content-Strategie (The Scripts)
- [ ] 15 Test-Szenarien aus echten Business-Apps und Games.

6. QA-Paket
- [ ] Test: Ist die Unterscheidung zwischen "Action" und "Expected Result" für den Spieler intuitiv?
- [ ] Test: Wirkt das Film-Theme motivierend für die eigentlich trockene Dokumentations-Arbeit?

7. Abnahme-Kriterien
- [ ] Der Spieler verankert: "Ein guter Testfall ist so genau, dass ihn jeder nachspielen kann".
- [ ] Das Interface wirkt kreativ, professionell und unterhaltsam.
- [ ] Fokus auf präzise technische Dokumentation.
