1. "Test-Case-Writer"
   - Ziel: Schreibe präzise Testbeschreibungen.
   - Spielmechanik: 
     - Was gehört rein? Titel, Vorbedingung, Schritte, Erwartung, Ist-Zustand.
     - Lernziel: Standardisierte Testfallgestaltung.

2. "Bug-Report-Puzzle"
   - Ziel: Erstelle einen Bug-Report aus einem Crash-Protokoll.
   - Spielmechanik: 
     - "Klick auf Save -> Error 500".
     - Baue den Report: Repro-Schritte + Stacktrace.
     - Lernziel: Fehlerdokumentation.

3. "Test-Suite-Selection"
   - Ziel: Wähle die richtigen Testfälle für ein Szenario.
   - Spielmechanik: 
     - "Wir ändern das Login". Welche Tests aus der Suite sind passend?
     - Lernziel: Testfallmanagement.

4. "Priority-Setter"
   - Ziel: Weise Prioritäten zu.
   - Spielmechanik: 
     - Blocker vs. Low Priority Fehler.
     - Lernziel: Triage im Testing.

5. "Data-Driven-Challenge"
   - Ziel: Nutze Variablen in Testfällen.
   - Spielmechanik: 
     - Ein Testcase, viele CSV-Zeilen mit verschiedenen Daten.
     - Lernziel: Skalierbarkeit von Tests.
