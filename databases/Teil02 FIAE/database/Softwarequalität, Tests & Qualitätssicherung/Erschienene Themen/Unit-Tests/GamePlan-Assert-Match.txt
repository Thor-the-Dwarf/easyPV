Anforderungen (UX-Game-Optimiert)
- Ziel: Erstellung hochgradig spezialisierter, isolierter Tests für kleinste Software-Einheiten (Funktionen/Methoden).
- Game-Mechanic: "The Lego-Block Test". In deiner digitalen Werkstatt baust du ein gigantisches Schiff. Jedes Bauteil (Die Funktion) wird einzeln auf den "Prüfstand" (Anforderungshöhe) gelegt. Du musst mit Laser-Sonden (Asserts) prüfen: Ist dieser Stein genau 4 Noppen breit (`assertEquals`)? Ist er nicht hohl (`assertNotNull`)? Nur perfekte Steine kommen in den Verbau.
- Challenge: 
  - Level 1 (The Single Stud): Teste eine `add(a, b)` Funktion. Was passiert bei Null?
  - Level 2 (The Broken Brick): Eine Funktion liefert einen falschen String. Finde den Fehler durch gezielte Asserts.
  - Level 3 (The Isolation-War): Die Komponente will heimlich auf die Datenbank (Dependency) zugreifen. "Mappe" (Mocke) den Zugriff ab, damit nur die Logik des Steins geprüft wird.

Arbeitspakete (UX-Game-Optimiert)
1. Interactive HUD und Scoring
- [ ] HUD-Elemente: "Isolation-Rank", "Assertion-Count", "Instant-Green-Light".
- [ ] Scoring: "Speed-Dev" Bonus für Tests, die in <1ms laufen. Abzug bei "Bloated-Tests" (zu viel Setup für zu wenig Logik).

2. Screen-Layout (The Testing Bench)
- [ ] `workbench -> the-brick-display -> assertion-toolbox -> test-runner-strip`.
- [ ] Workbench: Ein technischer Tisch mit blauen Markierungen für die Maße.
- [ ] Assertion-Toolbox: Eine Liste von Werkzeugen (`==`, `!=`, `>`, `NULL`).

3. Kernvisuals und Effekte
- [ ] Modern-Industrial Ästhetik: Saubere weiße Oberflächen, gelbe Warnmarkierungen, mechanische Greifarme, helle LEDs (Shared Theme kompatibel).
- [ ] "Assert-Zap": Ein kleiner Laserstrahl schießt auf den Stein. Wenn der Test passt, leuchtet er grün auf.
- [ ] "Fracture-Scan": Bei einem fehlgeschlagenen Assert zeigt der Stein einen virtuellen Riss.

4. Browser- und Device-Resilienz
- [ ] `flexbox` für die Werkzeugleiste am unteren Rand.
- [ ] Mobile: "Drag-and-Drop" der Assertions auf die Code-Zeilen.

5. Content-Strategie (The Parts)
- [ ] 20 Baustein-Challenges: Von "String-Formatierung" bis "Mathematische Algorithmen".

6. QA-Paket
- [ ] Test: Ist der "Instant-Green-Light" Effekt befriedigend genug (Gamification)?
- [ ] Test: Sind die Fehlermeldungen der Asserts (Expected vs. Actual) im HUD gut lesbar?

7. Abnahme-Kriterien
- [ ] Der Spieler begreift: Unittests müssen klein, schnell und isoliert sein.
- [ ] Das Interface wirkt aufgeräumt, effizient und technisch modern.
- [ ] Maximale Punkte durch präzise Asserts und sauberes Mocking.
