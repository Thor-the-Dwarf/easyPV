Anforderungen (UX-Game-Optimiert)
- Ziel: Vertiefung des Verständnisses von internen Programmabläufen und deren gezielte Absicherung durch Testabdeckung (Pfadabdeckung).
- Game-Mechanic: "The Path Navigator". Du steuerst eine winzige Drohne durch das Innere eines Computer-Chips. Der Quellcode ist ein Labyrinth aus Korridoren (Anweisungen) und Weichen (IF/ELSE). Deine Drohne hinterlässt eine grüne Spur (Coverage). Ziel: Fliege JEDEN Korridor mindestens einmal ab, ohne an eine geschlossene Weiche (Bedingung nicht erfüllt) zu knallen.
- Challenge: 
  - Level 1 (The Straight Line): Fliege eine einfache Funktion ohne Abzweigungen ab.
  - Level 2 (The Junction): Eine `IF-ELSE` Weiche. Wähle den richtigen Input, um beide Arme der Weiche nacheinander "grün" zu färben.
  - Level 3 (The Loop-Vortex): Eine `WHILE`-Schleife. Fliege 3 Runden und breche dann korrekt aus, um das Ende des Korridors zu erreichen.

Arbeitspakete (UX-Game-Optimiert)
1. Interactive HUD und Scoring
- [ ] HUD-Elemente: "Path-Coverage" (%), "Battery (Input-Complexity)", "Crystal-Clear Logic Index".
- [ ] Scoring: "Navigator-Bonus" für das Finden des kürzesten Weges zur 100% Pfadabdeckung. Abzug bei "Dead-Ends" (nicht erreichbarer Code).

2. Screen-Layout (The Chip-Interior)
- [ ] `cockpit-view -> source-code-blueprint -> drone-control -> coverage-heatmap`.
- [ ] Source-Code-Blueprint: Der Code wird als leuchtendes Adern-System im Hintergrund dargestellt.
- [ ] Heatmap: Bereits beflogene Pfade leuchten neon-grün, unberührte Pfade sind dunkelblau.

3. Kernvisuals und Effekte
- [ ] Sci-Fi Tron Ästhetik: Dunkler Hintergrund, leuchtende Vektor-Linien, digitale Glitch-Effekte, elektronischer Puls-Sound.
- [ ] "Line-Highlight": Wenn die Drohne eine Zeile passiert, blitzt diese kurz hellweiß auf und wird dann grün.
- [ ] "Condition-Snap": Wenn eine Bedingung (`true`/`false`) gewählt wird, klappt die Weiche mit einem mechanischen Sound um.

4. Browser- und Device-Resilienz
- [ ] `SVG` für das scharfe Rendern der Pfade auf allen Auflösungen.
- [ ] Mobile: "Tilt-to-Fly" (Gyroskop) oder On-Screen Joystick.

5. Content-Strategie (The Circuits)
- [ ] 14 Logik-Schaltungen: Von "Login-Prüfung" bis "Komplexer Rabatt-Algo".

6. QA-Paket
- [ ] Test: Sind die IF-Bedingungen (Input-Boxen) während des Fluges einfach zu bedienen?
- [ ] Test: Bleibt der Überblick über den "gesamten Code" auch bei Zoom-Ins erhalten?

7. Abnahme-Kriterien
- [ ] Der Spieler verankert: "White-Box heißt, man sieht das Innere der Logik".
- [ ] Das Interface wirkt futuristisch, rasant und technisch präzise.
- [ ] 100% Pfadabdeckung als ultimatives Ziel.
