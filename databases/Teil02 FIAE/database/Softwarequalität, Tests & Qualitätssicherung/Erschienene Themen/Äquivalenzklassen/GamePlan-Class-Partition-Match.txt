Anforderungen (UX-Game-Optimiert)
- Ziel: Reduzierung der Testfälle durch Bildung von Äquivalenzklassen und Grenzwertanalyse zur Steigerung der Test-Effizienz.
- Game-Mechanic: "The Group Sorter". Du arbeitest an der Sortieranlage eines Logistik-Zentrums. Tausende von Paketen (Inputs) prasseln auf dich ein. Du kannst nicht jedes Paket einzeln wiegen! Du musst sie in "Zonen" (Klassen) einteilen (z.B. "0-10kg", "11-50kg", "Ungültig"). Ein Testpaket pro Zone reicht aus, um das System zu prüfen.
- Challenge: 
  - Level 1 (The Weight Check): Bilde Äquivalenzklassen für eine Altersprüfung (0-17, 18-99, Ungültig).
  - Level 2 (The Border-Patrol): Die Grenzwertanalyse! Ein Ticket kostet 10€. Prüfe genau die Kanten: 9.99€, 10.00€, 10.01€. Funktioniert die Waage an den Schwellen?
  - Level 3 (The Multi-Dimension): Zwei Inputs gleichzeitig (z.B. Alter UND Budget). Bilde eine Matrix aus Klassen und finde die minimal nötige Anzahl an Testpaketen.

Arbeitspakete (UX-Game-Optimiert)
1. Interactive HUD und Scoring
- [ ] HUD-Elemente: "Testing-Efficiency-Index", "Logical-Clarity-Meter", "Redundancy-Warning".
- [ ] Scoring: "Efficiency-Pro" Bonus für die geringste Anzahl an Testfällen bei 100% Logik-Abdeckung. Abzug für "Testing-Overhead" (unnötige Doppel-Tests).

2. Screen-Layout (The Logistic Center)
- [ ] `conveyor-belt -> input-stream -> class-containers (Zonen) -> validation-stamp`.
- [ ] Conveyor-Belt: Pakete mit Werten (Zahlen, Strings) fahren von links nach rechts durch den Screen.
- [ ] Containers: Boxen, in die man die Pakete zieht, um eine Klasse zu "definieren".

3. Kernvisuals und Effekte
- [ ] Modern-Logistic Ästhetik: Gelbe Roboterarme, schwarz-gelbe Warnstreifen, digitale Anzeigen, futuristische Container.
- [ ] "Zone-Glow": Wenn ein Paket in die richtige Zone gezogen wird, leuchtet der Container-Rand kurz grün.
- [ ] "Efficiency-Spark": Bei jeder erfolgreichen Reduktion von Testfällen sprühen oben kleine goldene Funken.

4. Browser- und Device-Resilienz
- [ ] `Draggable API` oder `Pointer Events` für das Verschieben der Pakete.
- [ ] Mobile: "Tap-to-Assign" Mechanik.

5. Content-Strategie (The Manifests)
- [ ] 16 Test-Szenarien: Von "Passwort-Länge" bis "Temperatur-Sensor-Validierung".

6. QA-Paket
- [ ] Test: Wird das Konzept der Grenzwertanalyse (Border-Patrol) visuell deutlich hervorgehoben?
- [ ] Test: Ist das Handling der "Pakete" auf kleinen Screens flüssig genug?

7. Abnahme-Kriterien
- [ ] Der Spieler begreift: Weniger Tests können die gleiche Sicherheit bieten, wenn man klug wählt.
- [ ] Das Interface wirkt strukturiert, effizient und hochgradig logisch.
- [ ] Fokus auf strategische Testfall-Design-Techniken.
