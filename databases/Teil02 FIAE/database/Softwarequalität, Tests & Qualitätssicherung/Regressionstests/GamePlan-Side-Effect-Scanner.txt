Anforderungen (UX-Game-Optimiert)
- Ziel: Sicherstellen, dass neue Code-Änderungen keine bestehenden Funktionalitäten beeinträchtigen (Regressionstest).
- Game-Mechanic: "The Time Butler". Du bist der Wächter des "Gestern". Ein neues Feature (Das Kind) will ins Haus. Doch Vorsicht: Es könnte die alten Möbel (Bestehende Funktionen) zerschlagen. Du lässt das Feature nur rein, wenn alle alten "Sicherheitsschranken" (Automatisierte Tests) grün bleiben.
- Challenge: 
  - Level 1 (The Small Fix): Ein kleiner Bugfix wird eingespielt. Scanne das Haus – ist die Kaffeemaschine (Andere Funktion) noch heil?
  - Level 2 (The Feature-Bomb): Ein Riesen-Modul wird hinzugefügt. Der Butler muss 50 Tests gleichzeitig überwachen. Wo brennt es?
  - Level 3 (The Ghost-Bug): Eine Änderung an der Datenbank wirkt sich auf das Reporting aus. Finde den Seiteneffekt, bevor der Butler verzweifelt.

Arbeitspakete (UX-Game-Optimiert)
1. Interactive HUD und Scoring
- [ ] HUD-Elemente: "Legacy-Stability-Gauge", "Regression-Dschungel-Radar", "Active-Guard-Points".
- [ ] Scoring: "Bulletproof-Bonus" für Änderungen ohne einen einzigen Regressionsfehler. Abzug bei "Collateral-Damage" (Seiten-Effekte).

2. Screen-Layout (The Victorian Mansion)
- [ ] `grand-hallway -> feature-gate -> testing-chambers (The Suites) -> result-telegraph`.
- [ ] Feature-Gate: Hier klopfen die neuen Code-Pakete an.
- [ ] Testing-Chambers: Jede Kammer repräsentiert ein bestehendes Modul, das im Hintergrund "grün" leuchten muss.

3. Kernvisuals und Effekte
- [ ] Steampunk-Mansion Ästhetik: Holz-Paneele, kupferne Zahnräder, grüne Gaslampen (Pass), roter Dampf (Fail), tickende Uhren.
- [ ] "Radar-Ping": Bei jeder Änderung geht ein Schallwellen-Effekt durch das Haus. Alles, was rot aufblitzt, ist kaputt.
- [ ] "The Butler's Nod": Bei erfolgreichem Test nickt eine animierte Butler-Silhouette am Rand.

4. Browser- und Device-Resilienz
- [ ] `mix-blend-mode` für die Radar-Lichteffekte.
- [ ] Mobile: "Tap-to-Scan" für die schnelle Analyse.

5. Content-Strategie (The Library)
- [ ] 18 realistische Regressions-Szenarien: "CSS-Änderung bricht Formular", "Library-Update zerstört Auth".

6. QA-Paket
- [ ] Test: Ist das frustrierende Gefühl einer Regression ("Warum geht das jetzt nicht mehr?!") spielerisch gut eingefangen?
- [ ] Test: Bleibt das Steampunk-Design auch ohne Hardcoding (shared_theme.css) konsistent?

7. Abnahme-Kriterien
- [ ] Der Spieler begreift: Jede Änderung braucht einen Sicherheits-Scan.
- [ ] Das Interface wirkt nostalgisch, charmant und hochgradig interaktiv.
- [ ] Hoher Lerneffekt für CI/CD und Test-Automatisierung.
