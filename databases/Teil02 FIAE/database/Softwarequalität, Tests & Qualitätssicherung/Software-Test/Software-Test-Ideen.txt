1. "Test-Level-Sortierer"
   - Ziel: Ordne Testarten der Hierarchie zu.
   - Spielmechanik: 
     - Unit Test (unten) -> Integration Test -> System Test -> Acceptance Test (oben).
     - Bringe die Begriffe in die V-Modell Reihenfolge.
     - Lernziel: Teststufen-Modell.

2. "Bug-BINGO"
   - Ziel: Finde die häufigsten Fehlerquellen.
   - Spielmechanik: 
     - Bringe Beschreibungen mit Testarten zusammen: "Zusammenspiel zweier Module prüfen". -> Integrationstest.
     - Lernziel: Zweck der verschiedenen Testphasen.

3. "V-Modell-Puzzler"
   - Ziel: Verknüpfe Entwicklungsphasen mit Testphasen.
   - Spielmechanik: 
     - Anforderungsanalyse <-> Abnahmetest.
     - Implementierung <-> Modultest.
     - Lernziel: Verifikation vs. Validation.

4. "Fehler-Kosten-Rechner"
   - Ziel: Warum testen wir so früh?
   - Spielmechanik: 
     - Fehler in Analyse gefunden: 10€. 
     - Fehler nach Release gefunden: 10.000€.
     - Lernziel: Wirtschaftlichkeit von Softwaretests.

5. "Test-Glossar-Match"
   - Ziel: Lerne die Fachbegriffe.
   - Spielmechanik: 
     - "Regressionstest". -> Prüfen, ob Altes noch geht.
     - "Sanity Test". -> Erste grobe Funktionsprüfung.
     - Lernziel: Fachsprachliche Sicherheit.
