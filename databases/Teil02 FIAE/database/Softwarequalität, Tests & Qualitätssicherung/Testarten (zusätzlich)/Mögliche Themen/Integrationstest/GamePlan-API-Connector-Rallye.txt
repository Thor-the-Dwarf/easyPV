Anforderungen (reverse engineered)
- Ziel: Verifizierung der Schnittstellen (Interfaces) und des Zusammenspiels zwischen verschiedenen Modulen.
- Kontext: Integrationstests, API-Testing, JSON/XML-Validierung, Middleware.
- UX-Prinzip:
  - "The Digital Handshake": Zwei Roboterarme (Module) versuchen, sich die Hand zu geben. Wenn die Handform (Datenstruktur) nicht passt, scheitert der Kontakt.
  - "Protocol-Match": Der Spieler konfiguriert die Schnittstellen-Parameter, damit die Daten reibungslos fließen können.
  - "The Data-Stream": Visualisierung von Daten-Paketen, die von Modul A zu Modul B "reisen" und dabei auf Barrieren (Inkompatibilität) stoßen.

Arbeitspakete (UX-Global-Optimiert)
1. Informationsarchitektur und Fokus-Hierarchie
- [ ] Sichtprioritaeten: 1 = Der Daten-Flow (Unterwegs), 2 = Die Schnittstellen-Definition (Soll), 3 = Der Response-Header (Ist).
- [ ] Textbudget: Max. 5 Zeilen JSON-Fragment pro Request/Response.
- [ ] "Handshake-Meter": Zeigt an, wie harmonisch die Module interagieren (0-100%).

2. Screen-Layout (Container-Ebenen reduzieren)
- [ ] Struktur: `page -> network-tunnel -> api-terminals -> log-monitor`.
- [ ] Tunnel: Zentrale Röhre, durch die Datenpakete flitzen (Desktop horizontal).
- [ ] Terminals: Interaktive Eingabefelder für Modul A (Sender) und Modul B (Empfänger).

3. Browser- und Device-Resilienz
- [ ] `min-height: 100dvh` Standard.
- [ ] Mobile: "Tap-to-Inspect" für Datenpakete im Tunnel; Drag & Drop für Connector-Bausteine.
- [ ] `will-change: left` für flüssige Paket-Animationen auf allen Geräten.

4. Kernvisuals
- [ ] Cyber-Infrastruktur Ästhetik: Leuchtende Glasfaserkabel, binäre Muster, blinkende Status-LEDs (shared_theme.css).
- [ ] "The Crash": Visuelle Funkenbildung, wenn ein Paket auf ein inkompatibles Interface trifft.
- [ ] High-Contrast Code-Badges (GET, POST, 200 OK, 404 NOT FOUND).

5. Interaktion und Feedback
- [ ] Connect-Feedback: "Klick! Datentyp 'String' passt in das Feld 'Username'. Die Verbindung ist stabil."
- [ ] Fehler-Analyse: "Bzzz! Der Controller erwartet ein Datum im Format 'YYYY-MM-DD', du hast aber 'DD.MM.YYYY' gesendet. Handshake gescheitert!"
- [ ] Erfolg: "Brücke gebaut! Alle Module kommunizieren fehlerfrei miteinander."

6. KPI-Klarheit
- [ ] Interface-Accuracy: "Schnittstellen-Konformität: 100%".
- [ ] Data-Integrity: "Datenverlust im Exchange: 0%".

7. Content-/Copy-Feinschliff
- [ ] Szenarien: "Frontend meldet sich beim Login-Backend", "Bezahldienst sendet Webhook an Warenkorb", "Microservice-A ruft Stammdaten von Service-B ab".
- [ ] Tonalität: Technisch, präzise, vernetzt.

8. QA-Paket
- [ ] Screenshot-Check: Sind die JSON-Fragmente auf einem Handy-Display lesbar und editierbar?
- [ ] Test: Funktioniert die Simulation von Timeouts und Netzwerkverzögerungen erkennbar?

9. Abnahme-Kriterien
- [ ] Der Spieler versteht, dass "funktionierende Module" einzeln nichts über das Gesamtsystem aussagen.
- [ ] Das Interface wirkt wie ein Profi-Diagnose-Tool für Netzwerktechniker.
- [ ] Volle Beherrschung der Konzepte Request, Response und Payload.
