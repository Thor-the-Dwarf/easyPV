Anforderungen (reverse engineered)
- Ziel: Differenzierung der Projektphasen im UI-Design (Skizze -> Wireframe -> Mockup -> Prototyp).
- Kontext: Design-Workflow, Low-Fidelity vs. High-Fidelity, Kundenkommunikation, Iteration.
- UX-Prinzip:
  - "The Blueprint Evolution": Ein Gebäude entsteht nicht sofort farbig.
  - "Phase 1: Napkin Sketch" (Die Idee): Schnelle Striche, Fokus auf das grobe Konzept.
  - "Phase 2: The Blueprint" (Wireframe): Fokus auf Struktur und Wege, keine Farben (Graustufen).
  - "Phase 3: The Photo" (Mockup): Pixel-perfekt, Farben, Typografie, echtes Look-and-Feel.
  - "Phase 4: The House-Tour" (Prototyp): Interaktiv, Klicks funktionieren.

Arbeitspakete (UX-Global-Optimiert)
1. Informationsarchitektur und Fokus-Hierarchie
- [ ] Sichtprioritaeten: 1 = Das Design-Artefakt (Das Bild), 2 = Der Fachbegriff (Die Phase), 3 = Der Zweck des Artefakts.
- [ ] Textbudget: Max. 8 Wörter pro Phasenbeschreibung.
- [ ] "Fidelity-Slider": Visualisierung der Detailtiefe von 0 bis 100%.

2. Screen-Layout (Container-Ebenen reduzieren)
- [ ] Struktur: `page -> timelines-axis -> design-cards -> definition-bins`.
- [ ] Axis: Eine horizontale Zeitlinie mit den Projektphasen (Desktop oben).
- [ ] Cards: Hochwertige Bildbeispiele der Entwürfe in der Mitte.

3. Browser- und Device-Resilienz
- [ ] `min-height: 100dvh` Standard.
- [ ] Mobile: "Swipe-to-Categorize" Mechanik; Karten auf die richtige Stelle der Zeitlinie schieben.
- [ ] `aspect-ratio: 16/9` für die Design-Previews zur Vermeidung von Layout-Shifts.

4. Kernvisuals
- [ ] Architektur-Ästhetik: Pergament, Blaupausen-Hintergrund, CAD-Raster, elegante Typografie (shared_theme.css).
- [ ] "The Morph": Wenn eine Zuordnung stimmt, verwandelt sich der Wireframe kurzzeitig in das fertige Mockup (Animation).
- [ ] High-Contrast Badges für "Low-Fi" und "High-Fi".

5. Interaktion und Feedback
- [ ] Match-Feedback: "Klick! Korrekt. Im Wireframe klären wir die Anordnung der Elemente, ohne uns in Farben zu verlieren."
- [ ] Prozess-Tipp: "Achtung! Du willst eine detaillierte Grafik im ersten Meeting zeigen? Fang lieber mit einem Wireframe an, um die Struktur zu klären."
- [ ] Erfolg: "Design-Stratege! Du kennst jetzt den Weg von der ersten Skizze zum perfekten Produkt."

6. KPI-Klarheit
- [ ] Accuracy-Rate: "Phasen-Zuordnung: 100%".
- [ ] Project-Speed: "Zeitgewinn durch klaren Design-Workflow: Maximal".

7. Content-/Copy-Feinschliff
- [ ] Szenarien: "Kritik an der Farbe im Wireframe (Antipattern)", "Struktur-Check vor dem Coding", "First Draft".
- [ ] Tonalität: Kreativ, professionell, prozessorientiert.

8. QA-Paket
- [ ] Screenshot-Check: Sind die Unterschiede zwischen Wireframe und Mockup auf kleinen Previews deutlich erkennbar?
- [ ] Test: Ist die zeitliche Abfolge der Kategorien pädagogisch sinnvoll erklärt?

9. Abnahme-Kriterien
- [ ] Der Spieler begreift, warum man nicht sofort mit dem Pixel-Design beginnt.
- [ ] Das Interface wirkt wie ein exklusives Portfolio einer Top-Agentur.
- [ ] Absolute Sicherheit bei der Verwendung der Fachbegriffe (Lo-Fi, Hi-Fi, Wireframe, Mockup).
