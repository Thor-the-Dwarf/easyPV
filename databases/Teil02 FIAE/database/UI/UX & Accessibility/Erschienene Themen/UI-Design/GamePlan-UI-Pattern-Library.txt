Anforderungen (reverse engineered)
- Ziel: Korrekte Anwendung und Benennung von UI-Design-Patterns zur Erhöhung der Erwartungskonformität und Gebrauchstauglichkeit.
- Kontext: Design-Spezifikation, Pattern-Libraries, UI-Komponenten, Standardisierung.
- UX-Prinzip:
  - "The Wordbook of Interaction": UI-Patterns sind die Wörter einer Sprache, die jeder Nutzer versteht. Ein "Modal" ist ein Wort, ein "Breadcrumb" ein anderes.
  - "Pattern-Matching": Der Spieler sieht eine Nutzeraufgabe (z.B. "Führe den Nutzer Schritt für Schritt durch eine Anmeldung") und wählt das passende "Wort" (einen Wizard).
  - "Consistency-Check": Auswahl der Komponenten, die denselben "Dialekt" (Designsystem) sprechen.

Arbeitspakete (UX-Global-Optimiert)
1. Informationsarchitektur und Fokus-Hierarchie
- [ ] Sichtprioritaeten: 1 = Das UI-Pattern (Das Visuelle), 2 = Der Name (Der Begriff), 3 = Der Anwendungsfall (Wann nutzen?).
- [ ] Textbudget: Max. 12 Wörter pro Pattern-Erklärung.
- [ ] "Pattern-DNA": Ein kurzer Steckbrief pro Komponente (Vorteil, Nachteil, Beispiel).

2. Screen-Layout (Container-Ebenen reduzieren)
- [ ] Struktur: `page -> pattern-gallery -> use-case-prompt -> category-bins`.
- [ ] Gallery: Ein Grid aus interaktiven Elementen (Desktop 70% Breite).
- [ ] Prompt: Eine Aufgabenstellung am oberen Rand (Sticky Header).

3. Browser- und Device-Resilienz
- [ ] `min-height: 100dvh` Standard.
- [ ] Mobile: "Card-Flipping" - Bild vorne, Begriff hinten; Drag & Drop in die richtige Kategorie.
- [ ] `grid-template-columns: repeat(auto-fit, ...)` für automatisches UI-Stacking.

4. Kernvisuals
- [ ] Modern-Clean Ästhetik: Viel Weißraum (oder Dark-Gray), perfekte Abstände, klare Konturen, dezente Schatten (shared_theme.css).
- [ ] "The Glow": Bei korrekter Wahl leuchtet das Element in einer harmonischen Brand-Farbe auf.
- [ ] High-Contrast Icons für abstrakte Patterns (z.B. Hamburger-Menü).

5. Interaktion und Feedback
- [ ] Choice-Feedback: "Klick! Korrekt. Ein Accordion spart Platz bei vielen sekündären Informationen."
- [ ] Anti-Pattern Warnung: "Halt! Ein Modal in einem Modal in einem Modal zu öffnen, verwirrt den Nutzer. Nutze lieber einen anderen Flow."
- [ ] Erfolg: "UI-Sprachgenie! Dein Vokabular an Patterns ist jetzt so groß, dass dich jeder Nutzer blind versteht."

6. KPI-Klarheit
- [ ] Vocabulary-Score: "Menge beherrschter Patterns: X".
- [ ] UX-Consistency: "Erwartungskonformitäts-Level: Maximal".

7. Content-/Copy-Feinschliff
- [ ] Patterns: Tabs, Accordions, Modals, Breadcrumbs, Tooltips, Steppers, Carousel, Data Tables.
- [ ] Tonalität: Strukturiert, erklärend, ästhetisch.

8. QA-Paket
- [ ] Screenshot-Check: Sind die Bild-Beispiele im Grid auf dem Smartphone noch detailreich genug?
- [ ] Test: Sind die Anwendungsbeispiele eindeutig (keine zwei richtigen Lösungen)?

9. Abnahme-Kriterien
- [ ] Der Spieler verankert UI-Patterns als standardisierte Kommunikationsmittel.
- [ ] Das Interface wirkt wie ein exklusiver Styleguide eines Big-Tech-Unternehmens.
- [ ] Absolute Sicherheit bei der Benennung und dem Einsatz von Interface-Komponenten.
