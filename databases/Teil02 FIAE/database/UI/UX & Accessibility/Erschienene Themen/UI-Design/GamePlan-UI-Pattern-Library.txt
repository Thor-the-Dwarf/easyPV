Anforderungen (UX-Game-Optimiert)
- Ziel: Kenntnis gängiger UI-Pattern (Cards, Accordions, Modals, Breadcrumbs) und deren korrekte Anwendung zur Lösung von Standardproblemen im Interface-Design.
- Game-Mechanic: "The Component-Sifter". Du bist der Architekt eines Interface-Baukastens. Ein Förderband liefert dir Anwendungsfälle (z.B. "Zeige 50 Produkte übersichtlich an", "Frage eine Bestätigung zum Löschen ab"). Du hast 4 Trichter, die für verschiedene Patterns stehen (Card-Layout, Modal, List-View, Accordion). Wirf den Anwendungsfall in den richtigen Trichter! Wenn du das Löschen in ein Accordion packst, gibt es Matsch!
- Challenge: 
  - Level 1 (The Basics): Klassische Zuordnung von Patterns zu Standard-Szenarien.
  - Level 2 (The Mobile-Shift): Übertrage Desktop-Patterns auf Mobile. Aus einem Hover-Menu wird ein Burger-Menu oder eine Tab-Bar.
  - Level 3 (The Pattern-Breaker): Erkenne "Dark Patterns" (Tricks, die den User täuschen) und sortiere sie in den Abfalleimer aus.

Arbeitspakete (UX-Game-Optimiert)
1. Interactive HUD und Scoring
- [ ] HUD-Elemente: "Pattern-Confidence Meter", "Logic-Stream Speed", "Component-Usage Stats", "Intuition-Score".
- [ ] Scoring: "UI-Architect" Bonus für schnelle und logische Zuordnungen. Abzug für "Anti-Pattern Usage" (schlechte UX-Wahl).

2. Screen-Layout (The Factory Floor)
- [ ] `conveyor-belt -> task-cards -> pattern-funnels -> quality-control-log`.
- [ ] Task-Cards: Karten mit kurzen Problembeschreibungen, die übers Band laufen.
- [ ] Pattern-Funnels: Animierte Trichter mit großen Icons der UI-Elemente.

3. Kernvisuals und Effekte
- [ ] Bauhaus-Industrie Ästhetik: Primärfarben, geometrische Formen, metallische Texturen, mechanische Sounds (Shared Theme kompatibel).
- [ ] "Component-Build": Wenn ein Pattern korrekt gewählt wurde, baut sich das Element im Hintergrund kurz modular auf.
- [ ] "Jam-Effect": Bei Fehlern stoppt das Band mit einem mechanischen Quietschen.

4. Browser- und Device-Resilienz
- [ ] `CSS Flexbox` für das Layout des Förderbands.
- [ ] Mobile: "Swipe left/right/up/down" um Elemente in die Trichter zu schieben.

5. Content-Strategie (The Pattern-Catalog)
- [ ] 20 verschiedene Szenarien und 10 UI-Standard-Patterns.

6. QA-Paket
- [ ] Test: Sind die Erklärungen, *warum* ein Pattern passt, im Quality-Log einsehbar?
- [ ] Test: Ist die Geschwindigkeit des Bands motivierend, aber nicht überfordernd?

7. Abnahme-Kriterien
- [ ] Der Spieler beherrscht das Vokabular und die Logik moderner Benutzeroberflächen.
- [ ] Das Interface wirkt strukturiert, effizient und industriell-schick.
- [ ] Fokus auf Standardisierung und Wiederverwendbarkeit in der UI.
