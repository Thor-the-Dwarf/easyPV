Anforderungen (UX-Game-Optimiert)
- Ziel: Erstellung und Validierung von interaktiven Klick-Dummys zur Simulation von Benutzerabläufen (User Flows).
- Game-Mechanic: "The User-Flow-Flight". Du navigierst einen kleinen Papierflieger (den User) durch ein Labyrinth aus App-Screens. Jeder Screen hat "Hotspots" (Klick-Bereiche). Triffst du den richtigen, fliegt der Flieger zum nächsten Ziel. Triffst du eine Sackgasse oder fehlt die Verbindung, stürzt der Flieger ab. Dein Ziel: Baue die Verbindungen so, dass der User mit minimalen Klicks am Ziel ankommt!
- Challenge: 
  - Level 1 (The Linear Path): Verbinde "Login" -> "Dashboard" -> "Logout".
  - Level 2 (The Multi-Option): Erstelle Pfade für "Passwort vergessen" und "Registrieren". Sorg für einen "Zurück"-Button überall!
  - Level 3 (The Feedback-Loop): Der User macht einen Fehler. Baue eine Error-Page, die ihn sanft zurück in den Flow leitet.

Arbeitspakete (UX-Game-Optimiert)
1. Interactive HUD und Scoring
- [ ] HUD-Elemente: "Flow-Connectivity (%)", "Click-Distance-Meter", "Dead-End-Alert", "Success-Rate".
- [ ] Scoring: "Navigator-Bonus" für den kürzesten Klickpfad. Abzug für "Orphaned Screens" (Seiten ohne Link).

2. Screen-Layout (The Flow-Map)
- [ ] `screen-overview -> hotspot-editor -> link-lines -> simulation-mode`.
- [ ] Link-Lines: Gekrümmte Linien, die die Screens im Editor-Modus verbinden (Figma-Style).
- [ ] Simulation-Mode: Der Fullscreen-Modus, in dem man den Prototyp wirklich "fliegen" kann.

3. Kernvisuals und Effekte
- [ ] Blueprint-Paper Ästhetik: Kariertes Papier, blaue Tinte, kleine Animationen, die wie handgezeichnet wirken (Shared Theme kompatibel).
- [ ] "Hotspot-Glow": Wenn man im Simulationsmodus zögert, leuchten die interaktiven Bereiche dezent auf.
- [ ] "Page-Flip": Ein eleganter 3D-Umblätter-Effekt beim Wechsel zwischen Screens.

4. Browser- und Device-Resilienz
- [ ] `CSS Transitions` für flüssige Seitenübergänge.
- [ ] Mobile: Unterstützung von Gesten (Swipe left/right) für die Navigation.

5. Content-Strategie (The User-Journeys)
- [ ] 12 verschiedene User-Journeys: Von "Produktsuche" bis "Support-Ticket".

6. QA-Paket
- [ ] Test: Ist die grafische Verbindung der Screens intuitiv bedienbar?
- [ ] Test: Fühlt sich der Simulationsmodus wie eine echte Web-App an?

7. Abnahme-Kriterien
- [ ] Der Spieler begreift: Ein Prototyp muss atmen und reagieren, bevor die erste Zeile produktiver Code steht.
- [ ] Das Interface wirkt dynamisch, verspielt und dennoch technologisch fundiert.
- [ ] Fokus auf Interaktionsdesign und Benutzerführung.
