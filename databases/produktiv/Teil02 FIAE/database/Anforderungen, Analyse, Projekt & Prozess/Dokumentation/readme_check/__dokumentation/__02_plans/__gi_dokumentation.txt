1. "Doku-Zielgruppen-Match"
   - Ziel: Unterscheide Benutzerdokumentation von technischer Dokumentation.
   - Spielmechanik: 
     - "Installationsanleitung für Admins". -> Technische Doku.
     - "Erste Schritte für Endanwender". -> Benutzer-Doku.
     - "API-Referenz". -> Entwickler-Doku.
     - Messbarkeit: Korrekte Zuordnung der Inhalte.
   - Lernziel: Zielgruppenorientiertes Schreiben.

2. "README-Check"
   - Ziel: Erstelle eine vollständige README.md für ein Projekt.
   - Spielmechanik: 
     - Bausteine: Project Title, Description, Installation, Usage, License, Contact.
     - Was fehlt in einem Open-Source Projekt? -> "License".
   - Lernziel: Standard-Strukturen in der Software-Doku.

3. "Code-Kommentar-Profi"
   - Ziel: Schreibe sinnvolle Kommentare im Code.
   - Spielmechanik: 
     - Schlechter Kommentar: `x = x + 1; // x um eins erhöhen`.
     - Guter Kommentar: `retryCount++; // Versuche zählen bis zum Timeout`.
     - Messbarkeit: Mehrwert des Kommentars erkennen.
   - Lernziel: "Self-documenting Code" & Clean Code Prinzipien.

4. "Versionierungs-Simulator"
   - Ziel: Halte die Dokumentation aktuell.
   - Spielmechanik: 
     - Software-Update auf Version 2.0.
     - Welche Dokumente müssen aktualisiert werden? (Handbuch, Changelog, FAQ).
     - Messbarkeit: Konsistenz der Information über alle Dokumente.
   - Lernziel: Dokumentations-Lifecycle.

5. "Automatisierungs-Helper"
   - Ziel: Lerne Tools zur automatischen Doku-Generierung kennen.
   - Spielmechanik: 
     - Match: Java -> Javadoc, Python -> Sphinx/Docstrings, JS -> JSDoc.
     - Messbarkeit: Wahl des richtigen Tools für die Sprache.
   - Lernziel: Effizienz in der Dokumentation.
