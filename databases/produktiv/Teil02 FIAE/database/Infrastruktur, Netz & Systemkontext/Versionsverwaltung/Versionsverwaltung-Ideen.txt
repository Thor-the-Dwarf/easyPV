1. "Commit-Message-Filter"
   - Ziel: Schreibe gute Historien.
   - Spielmechanik: 
     - "Fix" vs. "feat: add user login validation". Welcher Commit hilft dem Team mehr?
     - Lernziel: Kommunikation im Team.

2. "Merge-Conflict-Solver"
   - Ziel: Löse Code-Überschneidungen auf.
   - Spielmechanik: 
     - Zwei Leute haben dieselbe Zeile geändert. Wähle die richtige Version aus.
     - Lernziel: Kollaboratives Arbeiten.

3. "Git-Flow-Simulator"
   - Ziel: Nutze Feature-Branches.
   - Spielmechanik: 
     - `feature/login` -> `develop` -> `main`. Bringe den Code sicher ins Release.
     - Lernziel: Branch Management.

4. "Time-Travel (Checkout/Revert)"
   - Ziel: Gehe zurück in der Zeit.
   - Spielmechanik: 
     - Ein Bug wurde eingeführt. Finde den "bad commit" und mache ihn rückgängig.
     - Lernziel: Fehlerbehebung durch Historie.

5. "Stage-the-Changes"
   - Ziel: Verstehe den Staging-Area (Index).
   - Spielmechanik: 
     - Wähle nur die Dateien aus, die wirklich in den Commit sollen.
     - Lernziel: Präzises Arbeiten mit Git.
