1. "Architecture-Puzzle (VM vs. Docker)"
   - Ziel: Verstehe den technischen Aufbau.
   - Spielmechanik: 
     - VM: Host-OS -> Hypervisor -> Guest-OS -> Bin/Libs -> App.
     - Container: Host-OS -> Docker-Engine -> Bin/Libs -> App (Kein Guest-OS!).
     - Lernziel: Overhead-Reduzierung.

2. "Boot-Time-Racer"
   - Ziel: Erlebe die Geschwindigkeit von Containern.
   - Spielmechanik: 
     - Wer ist schneller bereit? VM (Minuten) vs. Container (Sekunden).
     - Lernziel: Agilität in der Infrastruktur.

3. "Isolations-Check"
   - Ziel: Sicherheit vs. Effizienz.
   - Spielmechanik: 
     - Eine VM ist komplett isoliert. Container teilen sich den Kernel. Was ist sicherer?
     - Lernziel: Security-Abwägungen.

4. "Portability-Challenge"
   - Ziel: "Build once, run anywhere".
   - Spielmechanik: 
     - Packe die App in ein Docker Image. Läuft sie auf Windows, Linux und Mac? (Ja).
     - Lernziel: Standardisierung.

5. "Resource-Efficiency Match"
   - Ziel: Wie viele Instanzen passen auf einen Host?
   - Spielmechanik: 
     - 10 VMs brauchen 10x das OS. 10 Container brauchen nur 1x das OS.
     - Lernziel: Kosteneffizienz.
