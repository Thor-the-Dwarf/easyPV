Anforderungen (UX-Game-Optimiert)
- Ziel: Verständnis der internen Phasen eines Compilers (Lexikalische, Syntaktische, Semantische Analyse, Codegenerierung).
- Game-Mechanic: "The Code Forge". Du bist der Meisterschmied in der "Silizium-Schmiede". Ein roher Erz-Code-Block (Source) kommt rein. Du musst ihn durch vier Maschinen jagen: Den "Zertrümmerer" (Lexer -> Tokens), das "Grammatik-Gitter" (Parser -> AST), den "Sinn-Prüfer" (Semantik) und den "Binär-Stempel" (Generator).
- Challenge: 
  - Level 1 (Token-Rausch): Sortiere Zeichenketten in die richtigen Token-Behälter (`IDENTIFIER`, `KEYWORD`, `OPERATOR`).
  - Level 2 (Tree-Builder): Setze aus Tokens einen abstrakten Syntaxbaum (AST) zusammen. Passt das `+` zwischen zwei Zahlen?
  - Level 3 (Type-Clash): Die Semantik-Maschine schlägt Alarm! Behebe den Typfehler (z. B. String + Int), bevor das Binary geschmiedet wird.

Arbeitspakete (UX-Game-Optimiert)
1. Interactive HUD und Scoring
- [ ] HUD-Elemente: "Forge-Heat", "Process-Current-Stage", "Optimization-Level".
- [ ] Scoring: "Precision-Forge" Bonus für das korrekte Durchlaufen aller Phasen ohne Fehlversuche. Abzug bei "Crash-Dumps".

2. Screen-Layout (The Underground Forge)
- [ ] `lava-stream -> source-input -> machine-stations (The 4 Stages) -> binary-output`.
- [ ] Machine-Stations: Vier hintereinandergeschaltete Stationen, die den Code verarbeiten.
- [ ] Binary-Output: Ein digital schimmernder "Eisenbarren" aus Nullen und Einsen.

3. Kernvisuals und Effekte
- [ ] Industrial-Magma Ästhetik: Dunkle Metallfarben, glühende Lava, Dampf-Animationen, schwere mechanische Kurbeln.
- [ ] "Transformation-Pulse": Jede Station hat eine eigene Licht-Signatur (Lexer = Gelb, Parser = Blau, etc.).
- [ ] "Error-Sparks": Wenn der Parser einen Syntaxfehler findet, sprühen an der Station Funken und die Maschine hakt.

4. Browser- und Device-Resilienz
- [ ] `CSS Animations` für den Laufband-Effekt des Codes.
- [ ] Mobile: "Swipe-to-Next-Station" für den manuellen Vorwärtsschritt.

5. Content-Strategie (The Source Ore)
- [ ] 10 Code-Snippets in verschiedenen Sprachen: Von C bis Python.

6. QA-Paket
- [ ] Test: Ist die Abgrenzung zwischen Lexer (Wörter) und Parser (Sätze) im Spiel haptisch spürbar?
- [ ] Test: Bleibt das Schmiede-Feeling auch bei schnellen Token-Eingaben erhalten?

7. Abnahme-Kriterien
- [ ] Der Spieler versteht den Compiler als multi-stufigen Umwandlungsprozess.
- [ ] Das Interface wirkt wuchtig, atmosphärisch und technisch faszinierend.
- [ ] Hoher motivatorischer Anreiz durch den "Schmiede-Erfolg" am Ende.
