Anforderungen (UX-Game-Optimiert)
- Ziel: Spieler beherrschen das Exception-Handling (Try, Catch, Finally, Throw) zur Erstellung robuster Software.
- Game-Mechanic: "The Safety Net". Deine App ist ein Akrobat, der über ein Seil (Der Happy Path) läuft. Plötzlich bricht ein Seilstück (`NullPointerException`, `IndexOutOfBounds`). Du musst blitzschnell das Sicherheitsnetz (Den Catch-Block) an der richtigen Stelle aufspannen, damit die App nicht in den Abgrund (Crash) stürzt.
- Challenge: 
  - Level 1 (Der Sturz): Ein Fehler tritt auf. Wähle den richtigen Netz-Typ (Exception-Klasse), um den Akrobaten aufzufangen.
  - Level 2 (The Clean-Up): Egal ob Sturz oder Erfolg – die Ressourcen (Bühnenlichter) müssen ausgeschaltet werden (`Finally`). 
  - Level 3 (Chain-Reaction): Ein Fehler in einer tiefen Methode wird hochgeworfen (`rethrow`). Fange ihn dort ab, wo die User-Meldung erscheint.

Arbeitspakete (UX-Game-Optimiert)
1. Interactive HUD und Scoring
- [ ] HUD-Elemente: "System-Resilience" (Health), "Stack-Trace History", "Active-Guard-Zone".
- [ ] Scoring: "Recovery-Bonus" für sauberes Weiterlaufen nach einem Fehler. Abzug bei "Silent-Catch" (Leere Catch-Blöcke).

2. Screen-Layout (The Performance Canvas)
- [ ] `the-great-void -> tightrope (The Code Path) -> trap-doors (Exceptions) -> safety-log`.
- [ ] Tightrope: Ein Pfad aus Code-Zeilen. Der Curser (Akrobat) läuft drüber.
- [ ] Trap-Doors: Abschnitte, die bei Fehlern rot aufklappen.

3. Kernvisuals und Effekte
- [ ] Cirque-du-Code Ästhetik: Dunkle Bühne, goldene Strahler, Seidenschnüre (Datenfluss), rote Warnleuchten.
- [ ] "Catch-Snap": Wenn eine Exception gefangen wird, federt das Netz den Fall elastisch ab.
- [ ] "Crash-Explosion": Wenn kein Catch vorhanden ist, zerfällt die App-Anzeige in digitale Pixel-Asche.

4. Browser- und Device-Resilienz
- [ ] `linear-gradient` für das Seil-Leuchten.
- [ ] Mobile: "Tap-to-Deploy" Netz an der Gefahrenstelle.

5. Content-Strategie (The Hazards)
- [ ] 20 Fehler-Szenarien: "Database Offline", "Divide by Zero", "Invalid JSON", "Network Timeout".

6. QA-Paket
- [ ] Test: Ist der Unterschied zwischen "spezifischem Catch" (z. B. `IOException`) und "generischem Catch" (`Exception`) visuell belohnt?
- [ ] Test: Fühlt sich der `Finally` Block als notwendiger Abschluss haptisch richtig an?

7. Abnahme-Kriterien
- [ ] Der Spieler begreift: Exceptions sind keine Fehler, sondern Ausnahmezustände, die man managt.
- [ ] Das Interface wirkt extrem hochwertig, spannend und lehrreich.
- [ ] Keine Angst mehr vor dem Stack-Trace.
