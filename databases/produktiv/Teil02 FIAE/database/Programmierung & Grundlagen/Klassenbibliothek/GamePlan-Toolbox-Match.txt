Anforderungen (UX-Game-Optimiert)
- Ziel: Zuordnung der richtigen Entwicklerwerkzeuge zu den Phasen des Software-Lifecycles (Plan, Code, Build, Test, Deploy, Run).
- Game-Mechanic: "The Tool Master". Du bist der Werkstatt-Meister in einer High-Tech Softwareschmiede. Eine Flut von Problemen (Tasks) kommt rein. Du musst blitzschnell das richtige Werkzeug aus deiner "Dev-Toolbox" (Git, Docker, IDE, Jira, Jenkins, npm) wählen und auf den Task anwenden.
- Challenge: 
  - Level 1 (Essentials): "Code muss versioniert werden" -> Ziehe `Git` drauf. "Aufgabe planen" -> `Jira`.
  - Level 2 (The Automated Factory): "Build & Test automatisieren" -> `Jenkins`. "Umgebung isolieren" -> `Docker`.
  - Level 3 (Full-Cycle): Ein Task wandert durch alle Phasen. Wechsle das Werkzeug synchron zum Fortschritt, ohne den Rhythmus zu verlieren.

Arbeitspakete (UX-Game-Optimiert)
1. Interactive HUD und Scoring
- [ ] HUD-Elemente: "Lifecycle-Progress" (Circular), "Mastery-Level", "Efficiency-Multiplier".
- [ ] Scoring: "Chain-Reaction" Bonus für das richtige Werkzeug in Rekordzeit. Abzug bei "Tool-Mismatch" (z. B. Docker zum Versionieren nutzen).

2. Screen-Layout (The Workbench)
- [ ] `blueprint-table -> lifecycle-conveyor -> tool-belt -> terminal-display`.
- [ ] Conveyor: Tasks fahren auf einem Fließband vorbei, jede Station ist eine SDLC-Phase.
- [ ] Tool-Belt: Eine scrollbare Leiste mit hochauflösenden, gläsernen Tool-Icons.

3. Kernvisuals und Effekte
- [ ] Industrial-Cyberpunk Ästhetik: Rostfreier Stahl, leuchtende Hologramme, mechanische Soundeffekte, klares Raster-Layout.
- [ ] "Action-Snap": Wenn ein Tool auf einen Task trifft, rastet es mit einem mechanischen "Klock" ein.
- [ ] "Phase-Transition": Die Hintergrundbeleuchtung ändert sich je nach SDLC-Phase (Plan = Blau, Deploy = Raketen-Gelb).

4. Browser- und Device-Resilienz
- [ ] `drop-shadow` Effekte für die Tool-Icons zur Erzeugung von Tiefe.
- [ ] Mobile: "Swipe-Tools-to-Task" Steuerung, optimiert für den Daumen.

5. Content-Strategie (The Tool-Catalog)
- [ ] 20 reale Dev-Tools: Von Git und AWS bis hin zu SonarQube und Kubernetes.

6. QA-Paket
- [ ] Test: Sind die Kurzbeschreibungen der Tools (beim Hover) prägnant genug für den Lerneffekt?
- [ ] Test: Bleibt das Fließband bei 60 FPS auch bei hoher Task-Dichte?

7. Abnahme-Kriterien
- [ ] Der Spieler lernt das moderne Dev-Ökosystem und dessen Zusammenspiel kennen.
- [ ] Das Interface wirkt professionell, State-of-the-Art und motivierend.
- [ ] Schneller, fehlerfreier Durchlauf durch den gesamten Lifecycle.
