1. "Package-Manager-Puzzle"
   - Ziel: Nutze Tools wie npm, pip oder Maven.
   - Spielmechanik: 
     - Aufgabe: "Installiere Express". Befehl: `npm install express`.
     - Aufgabe: "Zeige veraltete Pakete". Befehl: `npm outdated`.
     - Lernziel: Tooling-Kompetenz.

2. "Version-Resolver"
   - Ziel: Verstehe SemVer (Semantic Versioning).
   - Spielmechanik: 
     - `^1.2.3` vs. `~1.2.3`. Welches Update wird installiert?
     - Patch (1.2.4), Minor (1.3.0) oder Major (2.0.0).
     - Lernziel: Versionskonflikte vermeiden.

3. "Dependency-Hell-Escape"
   - Ziel: Löse Konflikte zwischen verschiedenen Bibliotheken.
   - Spielmechanik: 
     - Lib A braucht `lodash 3`. Lib B braucht `lodash 4`.
     - Finde eine Lösung (Update oder Adapter).
     - Lernziel: Problembehebung bei komplexen Abhängigkeiten.

4. "Lockfile-Checker"
   - Ziel: Verstehe den Sinn von `package-lock.json`.
   - Spielmechanik: 
     - Warum ist das Build auf dem Server anders als lokal? -> Lockfile nicht gepusht.
     - Lernziel: Reproduzierbare Builds.

5. "Audit-Runner"
   - Ziel: Automatische Prüfung auf Sicherheitslücken.
   - Spielmechanik: 
     - `npm audit fix`. 
     - Lernziel: Proaktive Wartung von Abhängigkeiten.
