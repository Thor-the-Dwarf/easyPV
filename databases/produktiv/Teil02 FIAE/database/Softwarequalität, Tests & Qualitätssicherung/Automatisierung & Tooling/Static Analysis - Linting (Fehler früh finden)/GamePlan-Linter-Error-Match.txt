Anforderungen (UX-Game-Optimiert)
- Ziel: Spieler erkennen den Nutzen von statischer Code-Analyse (Linting) zur Vermeidung von Fehlern und Einhaltung von Coding-Standards.
- Game-Mechanic: "The Static Shield". Dein Code-Raumschiff fliegt durch ein Asteroidenfeld aus "Bad Practice" und "Syntax-Glitches". Dein Schutzschild (Der Linter) scannt den Weg. Du musst die Schilder so konfigurieren (Rules), dass sie die gefährlichen Brocken (Fehler) abwehren, ohne den Flugfluss unnötig zu stoppen.
- Challenge: 
  - Level 1 (The Strict Guard): Aktiviere Regeln wie `no-unused-vars` und `eqeqeq`. Bereinige den Code, bevor das Schiff beschädigt wird.
  - Level 2 (The Style Wars): Setze konsistente Formatierungs-Regeln (Klammern, Semikolons) durch, um die "Crew-Harmonie" (Team-Collaboration) zu sichern.
  - Level 3 (The Zero-Tolerance Zone): Fliege durch eine Zone, in der JEDER Linter-Warning den Flug abbricht. Finde den perfekten "Golden-Standard" Pfad.

Arbeitspakete (UX-Game-Optimiert)
1. Interactive HUD und Scoring
- [ ] HUD-Elemente: "Shield-Stability-Index", "Violation-Detected List", "Code-Purity (Cleanliness)", "Autofix-Energy".
- [ ] Scoring: "Clean-Code-Pilot" Bonus für das manuelle Beheben von Warnungen ohne Autofix. Abzug bei "Rule-Bypass" (Ignorieren durch Kommentare).

2. Screen-Layout (The Cockpit)
- [ ] `radar-view (Code-Stream) -> rule-dashboard -> diagnostic-console -> thruster-status`.
- [ ] Radar-View: Ein vertikaler Strom aus Code-Zeilen. Fehler werden rot/gelb umrandet.
- [ ] Dashboard: Hier schaltest du Regeln (`rules: { ... }`) ein und aus.

3. Kernvisuals und Effekte
- [ ] Spaceship-Terminal Ästhetik: Dunkler Hintergrund, grüne Schrift (Old-School Terminal), rote Warnlampen, Cockpit-Soundkulisse (Shared Theme kompatibel).
- [ ] "Shield-Pulse": Wenn der Linter einen Fehler findet, prallt ein virtueller Laser auf den Schutzschild ab.
- [ ] "Diagnostic-Beep": Ein prägnantes Tonsignal bei jeder gefundenen Verletzung.

4. Browser- und Device-Resilienz
- [ ] `overflow-y: hidden` für den Code-Stream zur Erzeugung des Flug-Effekts.
- [ ] Mobile: "Swipe-to-Fix" Mechanik für schnelle Korrekturen.

5. Content-Strategie (The Asteroids)
- [ ] 20 Linter-Szenarien (ESLint/Stylelint Stil): Von "Unreachable Code" bis "Prefer Const".

6. QA-Paket
- [ ] Test: Ist das Feedback ("Warum ist das ein Fehler?") im HUD informativ genug?
- [ ] Test: Fühlt sich der "Flug durch den Code" flüssig und kontrolliert an?

7. Abnahme-Kriterien
- [ ] Der Spieler verankert: "Automatisierte Prüfungen sparen Zeit und Nerven".
- [ ] Das Interface wirkt immersiv, technisch anspruchsvoll und rasant.
- [ ] Fokus auf präventive Qualitätssicherung.
