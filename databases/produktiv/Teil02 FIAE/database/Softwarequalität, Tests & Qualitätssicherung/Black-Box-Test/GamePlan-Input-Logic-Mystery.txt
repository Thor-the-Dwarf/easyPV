Anforderungen (UX-Game-Optimiert)
- Ziel: Durchführung von Funktionstests ohne Kenntnis der inneren Struktur/Quellcode (Black-Box-Testing).
- Game-Mechanic: "The Secret Box". Vor dir steht ein schwarzer Tresor (Das System). Du siehst nur die Knöpfe (Eingaben) und den Auswurfschlitz (Ausgaben). Du musst durch systematisches Testen herausfinden: Welche Kombination an Knöpfen führt zum Erfolg? Was passiert bei illegalen Versuchen?
- Challenge: 
  - Level 1 (Der Code-Knacker): Finde die richtige Eingabe für eine triviale Logik (z. B. Passwort-Check ohne den Code zu sehen).
  - Level 2 (The Boundary Shadow): Teste die Grenzwerte. Was passiert bei `Minus-Werten` oder `Unendlich`?
  - Level 3 (The Mystery Machine): Eine komplexe Logik hinter der Wand. Erstelle eine Map aus Inputs und Outputs, um das Verhaltensmuster zu entschlüsseln.

Arbeitspakete (UX-Game-Optimiert)
1. Interactive HUD und Scoring
- [ ] HUD-Elemente: "Logic-Discovery-Rate" (%), "Input-Efficiency", "Vulnerability-Found Counter".
- [ ] Scoring: "Smart-Explorer" Bonus für das Finden aller Edge-Cases mit minimalen Versuchen. Abzug für "Random-Spamming" (Planloses Drücken).

2. Screen-Layout (The Dark Room)
- [ ] `safe-front -> input-console -> result-chute -> analysis-board`.
- [ ] Safe-Front: Eine massive, schwarze Box-Animation in der Mitte.
- [ ] Analysis-Board: Eine Tafel, auf der deine bisherigen Test-Ergebnisse (Input/Output Paare) automatisch mit Kreide skizziert werden.

3. Kernvisuals und Effekte
- [ ] Minimalist-Noir Ästhetik: Tiefes Schwarz, harte Schatten, ein einzelner Lichtkegel von oben, mechanische Klick-Geräusche.
- [ ] "Output-Slide": Bei jeder Eingabe schießt ein kleiner Zettel mit dem Resultat aus dem Schlitz.
- [ ] "Light-Up": Wenn du ein logisches Muster knackst, leuchtet ein Teil des Tresors von innen heraus auf.

4. Browser- und Device-Resilienz
- [ ] `box-shadow` und `radial-gradient` für die dramatische Beleuchtung.
- [ ] Mobile: "Dial-the-Inputs" Mechanik per Drehrad.

5. Content-Strategie (The Vaults)
- [ ] 12 geheimnisvolle Logik-Module: Von "Rabatt-Rechner" bis "Kredit-Check".

6. QA-Paket
- [ ] Test: Ist die Spannung ("Was ist da drin?") über das ganze Spiel spürbar?
- [ ] Test: Hilft das "Analysis-Board" dem Spieler, den Überblick über seine Test-Historie zu behalten?

7. Abnahme-Kriterien
- [ ] Der Spieler begreift: Testen geht auch ohne den Code zu lesen.
- [ ] Das Interface wirkt mysteriös, fokussiert und state-of-the-art.
- [ ] Hoher Drang zum "Logik-Enträtseln".
