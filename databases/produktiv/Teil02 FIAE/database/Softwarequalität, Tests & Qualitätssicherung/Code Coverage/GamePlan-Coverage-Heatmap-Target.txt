Anforderungen (UX-Game-Optimiert)
- Ziel: Spieler verstehen die Aussagekraft und die Grenzen der Testabdeckung (Line/Branch Coverage).
- Game-Mechanic: "The Heatmap Hunter". Du bist ein Wärmebild-Spezialist in einer "Eis-Festung" (Dem Code). Wo keine Tests laufen, herrscht klirrende Kälte (Blauer Code). Ziel ist es, durch kluge Test-Szenarien das "Feuer der Verifikation" zu entfachen, bis die ganze Festung in wohligem Grün (Wärme) erstrahlt.
- Challenge: 
  - Level 1 (Defrosting): Erziele 50% Abdeckung durch einen einfachen Testfall.
  - Level 2 (The Branch-Glacier): Ein riesiger Eisblock (komplexe IF-Struktur) versperrt den Weg. Finde die Inputs, um die "innere Wärme" freizusetzen.
  - Level 3 (The 99% Trap): Die letzten zwei Zeilen sind extrem schwer zu erreichen. Lohnt sich der Aufwand (Aufwand vs. Sicherheit)? Entscheide strategisch.

Arbeitspakete (UX-Game-Optimiert)
1. Interactive HUD und Scoring
- [ ] HUD-Elemente: "Main-Thermal-Meter" (Gesamt-Coverage %), "Uncovered-Hotspots", "Testing-Energy".
- [ ] Scoring: "Heat-Master" Bonus für das Erreichen von 80%+ Coverage in Rekordzeit. "Diminishing-Returns-Warning" bei 95%+.

2. Screen-Layout (The Frozen Fortress)
- [ ] `overview-map -> code-scroll-pane -> thermal-scanner-view -> coverage-bar`.
- [ ] Thermal-Scanner-View: Der Code verändert seine Farbe dynamisch von Blau (0%) über Gelb zu Grün (100%).
- [ ] Hotspots: Rot blinkende Symbole an besonders kritischen (aber ungetesteten) Stellen.

3. Kernvisuals und Effekte
- [ ] Frost-to-Forest Ästhetik: Eis-Kristalle auf dem Screen, die bei Fortschritt schmelzen und grünem Moos/Pflanzen (Code) weichen.
- [ ] "Thaw-Sound": Ein sanftes Knistern und Glucksen, wenn Code-Teile "aufgetaut" (abgedeckt) werden.
- [ ] "Steam-Effect": Wenn ein Test erfolgreich durchläuft, steigt virtueller Dampf vom Code auf.

4. Browser- und Device-Resilienz
- [ ] `CSS Filter (blur, brightness)` für den Frost-Effekt über dem Div.
- [ ] Mobile: "Tap-to-Heat" Mechanik zur Auswahl der Test-Daten.

5. Content-Strategie (The Cold Files)
- [ ] 15 Code-Dateien in verschiedenen Sprachen: Von "Utility-Lib" bis "Backend-Route".

6. QA-Paket
- [ ] Test: Ist der visuelle Kontrast zwischen "Kalt" und "Warm" stark genug für Barrierefreiheit?
- [ ] Test: Wirkt das "Schmelzen" des Eises motivierend für den Spieler?

7. Abnahme-Kriterien
- [ ] Der Spieler verankert: "Coverage zeigt, was NICHT getestet wurde (Lücken füllen)".
- [ ] Das Interface wirkt atmosphärisch, organisch und beeindruckend.
- [ ] Hoher motivatorischer Anreiz durch die "Auftau-Mechanik".
