1. "Peer-Review-Simulator"
   - Ziel: Finde Fehler im Code anderer Leute.
   - Spielmechanik: 
     - Du bist der "Reviewer". Hinterlasse Kommentare zu "Magic Numbers", "Duplikation", "Fehlenden Tests".
     - Lernziel: Zusammenarbeit und Qualitäts-Standards.

2. "Checkstyle-Match"
   - Ziel: Prüfe Einhaltung von Konventionen.
   - Spielmechanik: 
     - "CamelCase verletzt?", "Klammern falsch?".
     - Lernziel: Konsistenz in der Codebase.

3. "Review-Ego-Game"
   - Ziel: Lerne konstruktive Kritik zu geben und anzunehmen.
   - Spielmechanik: 
     - Wähle die freundlichste Art, einen Fehler anzumerken. 
     - Lernziel: Soft Skills in der Entwicklung.

4. "Pull-Request-Fieber"
   - Ziel: Verwalte mehrere Änderungen gleichzeitig.
   - Spielmechanik: 
     - Welche PRs können gemerged werden, welche brauchen Korrekturen?
     - Lernziel: Workflow in Git/GitHub.

5. "Pair-Programming-Rallye"
   - Ziel: Verstehe den "Vier-Augen-Prinzip" Vorteil.
   - Spielmechanik: 
     - Einer tippt (Driver), einer denkt (Navigator). 
     - Tausche die Rollen zeitgerecht.
     - Lernziel: Kontinuierliches Review.
