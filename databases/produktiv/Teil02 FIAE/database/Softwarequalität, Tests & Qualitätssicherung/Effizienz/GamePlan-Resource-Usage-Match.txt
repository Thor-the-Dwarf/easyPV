Anforderungen (UX-Game-Optimiert)
- Ziel: Identifikation und Behebung von Ressourcenverschwendung (CPU, RAM, Netzwerk) zur Steigerung der Software-Effizienz.
- Game-Mechanic: "The Resource Miser". Du bist der knallharte Buchhalter in einem Rechenzentrum. Jedes Byte kostet Gold! Deine App verbraucht zu viele Ressourcen. Du musst den Code scannen und "Verschwender" (z. B. unnötige Schleifen, Speicherleaks, fette Bilder) finden und eliminieren.
- Challenge: 
  - Level 1 (The Memory Leak): Finde ein Objekt, das niemals aus dem Speicher gelöscht wird. "Clowne" es weg, bevor der RAM überläuft.
  - Level 2 (The CPU-Hog): Eine Schleife berechnet 1000x das Gleiche. Nutze Caching (Memoization), um die CPU-Zyklen zu halbieren.
  - Level 3 (The Payload-Diet): Ein API-Call sendet 10MB Daten, obwohl nur 1KB gebraucht wird. Stutze das JSON-Objekt auf das Minimum.

Arbeitspakete (UX-Game-Optimiert)
1. Interactive HUD und Scoring
- [ ] HUD-Elemente: "System-Health (Temp)", "RAM-Usage-Bar", "CPU-Pressure", "Waste-Detected Count".
- [ ] Scoring: "Efficiency-Whiz" Bonus für die maximale Einsparung bei minimalem Code-Eingriff. Abzug für "Performance-Regressionen".

2. Screen-Layout (The Server Rack)
- [ ] `rack-monitor -> code-thermoscan -> resource-profiler (Charts) -> optimization-workbench`.
- [ ] Code-Thermoscan: Code-Teile leuchten orange/rot, wenn sie ineffizient sind.
- [ ] Resource-Profiler: Live-Graphen, die nach jeder Optimierung sinken.

3. Kernvisuals und Effekte
- [ ] Cyber-Clean Ästhetik: Weißes Licht, gläserne Panels, fließende blaue Datenströme, Alarm-Rot bei Overload, kühles Interface (Shared Theme kompatibel).
- [ ] "Heat-Dissipation": Wenn ein Fehler gefixt wird, entweicht virtueller Dampf und die Farbe wechselt von Rot zu Blau.
- [ ] "Efficiency-Sparkle": Ein sanftes Schimmern über dem optimierten Code-Block.

4. Browser- und Device-Resilienz
- [ ] `Intersection Observer` für das flüssige Nachladen der Profiler-Daten.
- [ ] Mobile: "Tap-to-Optimize" auf die roten Hotspots.

5. Content-Strategie (The Waste-Logs)
- [ ] 15 Ineffizienz-Fälle: Von "Nested Loops" bis "Redundant Database Queries".

6. QA-Paket
- [ ] Test: Fühlt sich der Zeitdruck durch die "RAM-Usage-Bar" motivierend an?
- [ ] Test: Sind die Einsparungen (z. B. "20% faster") visuell befriedigend aufbereitet?

7. Abnahme-Kriterien
- [ ] Der Spieler verankert: "Performance ist ein Feature".
- [ ] Das Interface wirkt modern, technologisch und hochgradig reaktiv.
- [ ] Fokus auf nachhaltige und ressourcenschonende Programmierung.
