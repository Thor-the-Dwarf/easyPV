1. "Side-Effect-Scanner"
   - Ziel: Finde Fehler, die durch Änderungen an anderen Stellen entstanden sind.
   - Spielmechanik: 
     - Du fixst einen Bug in Modul A. Plötzlich bricht Modul B. 
     - Finde den Zusammenhang.
     - Lernziel: Abhängigkeiten und Regressionrisiken.

2. "Test-Automation-Racer"
   - Ziel: Führe Tests schnell und wiederholt aus.
   - Spielmechanik: 
     - Baue eine CI/CD Pipeline, die bei jedem Commit alle alten Tests automatisch prüft.
     - Lernziel: Effizienz durch Automatisierung.

3. "Version-Comparison"
   - Ziel: Vergleiche das Verhalten von Version 1.0 und 1.1.
   - Spielmechanik: 
     - Hat sich die Performance eines alten Moduls verschlechtert?
     - Lernziel: Qualitätssicherung über Produktversionen hinweg.

4. "Legacy-Safety-Net"
   - Ziel: Schütze alten Code vor neuen Fehlern.
   - Spielmechanik: 
     - Schreibe Tests für eine bestehende Funktion, bevor du sie anfasst.
     - Lernziel: Refactoring-Sicherheit.

5. "Regression-Prioritizer"
   - Ziel: Wähle die wichtigsten Tests für einen schnellen Re-Run.
   - Spielmechanik: 
     - Wir haben nur 10 Minuten Zeit. Welche 5 Tests sind am kritischsten?
     - Lernziel: Selektive Regressionstests.
