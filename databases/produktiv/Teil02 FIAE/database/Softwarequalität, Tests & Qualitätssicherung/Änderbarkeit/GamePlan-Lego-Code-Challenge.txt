Anforderungen (UX-Game-Optimiert)
- Ziel: Verbesserung der Code-Struktur zur Steigerung der Wartbarkeit und zukünftigen Änderbarkeit (Refactoring).
- Game-Mechanic: "The Lego Refurbisher". Du hast ein altes, verkrustetes Lego-Modell (Legacy Code) geerbt. Der Kunde will eine neue Etage, aber alles ist zusammengeklebt (Hohe Kopplung) und unbeschriftet. Du musst Steine lösen, sie säubern (Refactoring), logisch gruppieren und so wieder zusammensetzen, dass Erweiterungen ein Kinderspiel werden.
- Challenge: 
  - Level 1 (The Glue-Trap): Löse eine hart-codierte Abhängigkeit auf. Ersetze sie durch ein flexibles Bauteil.
  - Level 2 (The Identity-Crisis): Benenne 5 Steine um, damit man sofort weiß, was sie tun (Naming).
  - Level 3 (The Modular Expansion): Baue das Modell so um, dass man eine neue Funktion in <10 Sekunden anstecken kann.

Arbeitspakete (UX-Game-Optimiert)
1. Interactive HUD und Scoring
- [ ] HUD-Elemente: "Technical-Debt-Meter", "Change-Agility Index", "Refactor-Progress", "Decoupling-Level".
- [ ] Scoring: "Clean-Code-Pro" Bonus für die Reduktion von technischer Schuld. Abzug bei "Breaking-Changes" (wenn das Modell beim Umbau zusammenbricht).

2. Screen-Layout (The Workshop)
- [ ] `model-view (The Legacy Code) -> separator-tool -> renaming-station -> component-shelf`.
- [ ] Model-View: Ein "wirrer Haufen" aus Code-Blöcken, die per Linien verbunden sind (Spaghetti-Graph).
- [ ] Component-Shelf: Hier legst du deine frisch "gewaschenen" (refactoreten) Module ab.

3. Kernvisuals und Effekte
- [ ] Workshop-Utility Ästhetik: Werkstatt-Farben (Grau, Orange, Blau), praktische Symbole, das Geräusch von klickenden Steinen, helle Ausleuchtung.
- [ ] "Clean-Glow": Wenn ein Code-Block refactored wurde, verliert er seinen "Dreck" (graue Textur) und wird leuchtend bunt.
- [ ] "Magnetic-Snap": Beim Zusammenbau ziehen sich saubere Module fast von selbst an.

4. Browser- und Device-Resilienz
- [ ] `Pointer Events` für das präzise Greifen und Ziehen der Code-Steine.
- [ ] Mobile: "Double-Tap to Refactor".

5. Content-Strategie (The Legacy-Models)
- [ ] 12 "kaputte" Code-Szenarien aus echten (fiktiven) Schrott-Projekten.

6. QA-Paket
- [ ] Test: Ist der Vorher/Nachher-Effekt (Technikschuld sinkt) motivierend genug?
- [ ] Test: Fühlt sich das "Auseinandernehmen" des Codes zerstörerisch oder befreiend an?

7. Abnahme-Kriterien
- [ ] Der Spieler verankert: "Wartbarkeit spart Zeit in der Zukunft".
- [ ] Das Interface wirkt praktisch, anfassbar und motivierend.
- [ ] Fokus auf saubere Software-Evolution.
