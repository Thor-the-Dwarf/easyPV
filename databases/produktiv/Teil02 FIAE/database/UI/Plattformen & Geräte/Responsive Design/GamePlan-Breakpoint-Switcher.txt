Anforderungen (UX-Game-Optimiert)
- Ziel: Beherrschung von responsivem Webdesign, Breakpoints und flexiblen Layout-Techniken (Flexbox, Grid) für verschiedene Endgeräte.
- Game-Mechanic: "The Screen-Shifter". Du bist der Zeremonienmeister einer digitalen Modenschau. Die Bühne (der Browser) ändert ständig ihre Breite! Deine Models (die UI-Elemente wie Header, Karten, Footer) müssen sich blitzschnell anpassen. 
  - Mal ist die Bühne schmal (Mobile): Models müssen sich stapeln.
  - Mal ist sie breit (Desktop): Models dürfen nebeneinander flanieren.
  Wenn ein Element aus der Bühne fällt oder sich unschön überlappt, gibt es Buh-Rufe. Setze die Breakpoints richtig!
- Challenge: 
  - Level 1 (The Stack-Up): Ordne 3 Karten so an, dass sie auf Mobilgeräten untereinander und auf Desktops nebeneinander stehen.
  - Level 2 (The Menu-Morph): Verwandle eine horizontale Navigationsleiste bei 600px in ein Burger-Menü.
  - Level 3 (The Multi-Device Challenge): Optimiere ein komplexes Dashboard für Smartphone, Tablet und 4K-Monitor gleichzeitig.

Arbeitspakete (UX-Game-Optimiert)
1. Interactive HUD und Scoring
- [ ] HUD-Elemente: "Current-Width (px)", "Active-Breakpoint Indicator", "Layout-Stability (%)", "Fluidity-Gauge".
- [ ] Scoring: "Flex-Master" Bonus für Layouts ohne horizontalen Scrollbalken. Abzug für "Overlapping Elements".

2. Screen-Layout (The Resize-Stage)
- [ ] `resize-handle -> layout-preview -> css-breakpoint-editor -> mobile-first-switch`.
- [ ] Resize-Handle: Ein Griff am rechten Rand, mit dem man den Viewport stufenlos schrumpfen oder dehnen kann.
- [ ] Mobile-First-Switch: Ein Modus, der den Spieler zwingt, erst das kleinste Layout zu definieren.

3. Kernvisuals und Effekte
- [ ] Futuristic-Elastic Ästhetik: Flexible Gitterlinien, fließende Animationen beim Resizen, Blueprint-Elemente in Electric Blue (Shared Theme kompatibel).
- [ ] "Snap-Effect": Wenn ein Breakpoint erreicht wird, gibt es ein subtiles "Klick"-Geräusch und das Layout springt in die neue Form.
- [ ] "Overflow-Warning": Wenn Text aus einer Box quillt, fängt die Box an, rot zu pulsieren.

4. Browser- und Device-Resilienz
- [ ] `CSS @media` Queries Simulation im Browser-Interface.
- [ ] Einsatz von `vmin`, `vmax` und `%` statt fester Pixelwerte.

5. Content-Strategie (The Viewport-Missions)
- [ ] 12 verschiedene Layout-Aufgaben: Von "Foto-Galerie" bis "Daten-Tabelle".

6. QA-Paket
- [ ] Test: Verhält sich das Layout beim manuellen Resizen absolut flüssig?
- [ ] Test: Wird der Vorteil von "Mobile First" durch die Level-Struktur klar?

7. Abnahme-Kriterien
- [ ] Der Spieler beherrscht das Denken in relativen Einheiten und Umbruchpunkten.
- [ ] Das Interface wirkt technisch brillant, reaktiv und dynamisch.
- [ ] Fokus auf geräteübergreifende Konsistenz und technische Flexibilität.
