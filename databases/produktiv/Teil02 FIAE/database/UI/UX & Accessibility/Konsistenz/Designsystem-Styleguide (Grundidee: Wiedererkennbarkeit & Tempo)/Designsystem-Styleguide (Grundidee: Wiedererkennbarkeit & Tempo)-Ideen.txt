1. "Component-Library-Match"
   - Ziel: Erkenne wiederverwendbare UI-Elemente.
   - Spielmechanik: 
     - "Primär-Button", "Sekundär-Button", "Eingabefeld", "Checkbox".
     - Ordne sie einem einheitlichen Designsystem zu.
     - Lernziel: Konsistenz durch Standardisierung.

2. "Token-Tracker"
   - Ziel: Nutze Design-Tokens für Farben und Abstände.
   - Spielmechanik: 
     - Ersetze `#FF0000` durch `$color-error`.
     - Lernziel: Skalierbarkeit von Designsystemen.

3. "Consistency-Detective"
   - Ziel: Finde "Outliers" in einer App.
   - Spielmechanik: 
     - "Warum hat dieser Button eine andere Rundung als alle anderen?".
     - Lernziel: Einhaltung von Styleguides.

4. "Atomic-Design-Puzzle"
   - Ziel: Baue komplexe Organismen aus Atomen und Molekülen.
   - Spielmechanik: 
     - Atom (Button) + Atom (Label) -> Molekül (Suchen-Feld).
     - Lernziel: Brad Frosts Atomic Design Prinzip.

5. "UI-Pattern-Quiz"
   - Ziel: Wähle das richtige Pattern für einen Anwendungsfall.
   - Spielmechanik: 
     - "Für eine Auswahl aus 3 Optionen: Radio Buttons oder Select Box?".
     - Lernziel: Best Practices für UI-Komponenten.
