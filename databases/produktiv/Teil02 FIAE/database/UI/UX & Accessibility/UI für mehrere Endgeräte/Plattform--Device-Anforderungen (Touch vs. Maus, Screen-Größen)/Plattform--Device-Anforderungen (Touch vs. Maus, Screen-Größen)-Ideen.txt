1. "Touch-Target-Master"
   - Ziel: Optimiere die Größe von interaktiven Flächen für Touch-Eingaben.
   - Spielmechanik: 
     - Klicke auf kleine Buttons mit dem Finger (simuliert: Großer Cursor).
     - Wenn die Buttons < 44x44px sind -> Fehlklick.
     - Lernziel: Ergonomische Mindestgrößen für mobile UIs.

2. "Hover-vs-Touch-Match"
   - Ziel: Kenne die Unterschiede in den Interaktionsmöglichkeiten.
   - Spielmechanik: 
     - "Welches Menü funktioniert nicht auf dem iPad?" (Menü, das nur auf Hover öffnet).
     - Lernziel: Plattformspezifische Interaktionsmuster.

3. "Input-Type-Switcher"
   - Ziel: Nutze die richtigen Keyboard-Layouts auf Mobile.
   - Spielmechanik: 
     - "E-Mail Feld". Welches Keyboard soll kommen? (E-Mail Layout mit @).
     - "Telefonnummer". Welches Keyboard? (Nummernblock).
     - Lernziel: Mobile Formularoptimierung.

4. "Screen-Size-Scanner"
   - Ziel: Reagiere auf das Seitenverhältnis (Portrait vs Landscape).
   - Spielmechanik: 
     - Drehe das Tablet. Wie ordnen sich die Elemente neu an?
     - Lernziel: Dynamische Layout-Anpassung.

5. "Performance-Low-Bandwidth-Quest"
   - Ziel: Optimiere für mobile Netzwerke.
   - Spielmechanik: 
     - Lade große Bilder vs. WebP. Wie lange dauert es bei 3G?
     - Lernziel: Performance als UX-Faktor.
