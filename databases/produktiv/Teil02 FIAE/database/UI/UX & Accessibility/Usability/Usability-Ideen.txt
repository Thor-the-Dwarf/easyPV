1. "Heuristik-Hunted"
   - Ziel: Finde Verstöße gegen Nielsens Heuristiken.
   - Spielmechanik: 
     - "Kein Zurück-Button". -> Verstoß gegen Nutzerkontrolle.
     - "Fehlermeldung: Error 0x823". -> Verstoß gegen Verständlichkeit.
     - Lernziel: Usability-Regelwerk.

2. "Mental Model Match"
   - Ziel: Entspricht das System der Erwartung des Users?
   - Spielmechanik: 
     - Wo erwartet der User das "X" zum Schließen? (Oben rechts).
     - Platziere es woanders und beobachte die Frustration (Simulierte Zeit).
     - Lernziel: Konventionen und Nutzererwartungen.

3. "Affordance-Entdecker"
   - Ziel: Sieht ein Element klickbar aus?
   - Spielmechanik: 
     - Link ohne Unterstreichung vs. Button mit Schatten.
     - Wer wird öfter geklickt?
     - Lernziel: Visuelle Hinweise auf Funktionen.

4. "Search-vs-Browse"
   - Ziel: Navigationswege optimieren.
   - Spielmechanik: 
     - Finde Produkt X. Einmal über die Suche, einmal über 10 Kategorien.
     - Welcher Weg ist effizienter für wen?
     - Lernziel: Information architecture.

5. "Wartezeit-Überbrücker"
   - Ziel: UX bei Latenz erhalten.
   - Spielmechanik: 
     - Backend braucht 3 Sekunden. 
     - Was hilft? -> Skeleton Screen vs. Sanduhr vs. Nichts tun.
     - Lernziel: Systemstatus-Sichtbarkeit.
