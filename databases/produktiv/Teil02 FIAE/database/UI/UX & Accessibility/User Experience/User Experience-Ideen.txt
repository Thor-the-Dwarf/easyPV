1. "Persona-Puzzle"
   - Ziel: Entwicklung von empathischem Design durch Personas.
   - Spielmechanik: 
     - Du hast eine Zielgruppe (z.B. "Rentner", "Gamer", "Business-Profi").
     - Ordne Features den Bedürfnissen zu. Wer braucht "Große Schrift"? Wer braucht "High Performance"?
     - Lernziel: Zielgruppenorientierung.

2. "Job-to-be-Done Switcher"
   - Ziel: Verstehe die Motivation der Nutzer.
   - Spielmechanik: 
     - "Ich will ein Ticket kaufen" vs "Ich will pünktlich ankommen". 
     - Wie verändert das das Design?
     - Lernziel: User Needs vs. Feature Requests.

3. "Emotional-UX-Meter"
   - Ziel: Beurteile die emotionale Wirkung.
   - Spielmechanik: 
     - Ein Formular mit 50 Feldern. Wie fühlt sich der User? (Genervt).
     - Ein "Ein-Klick-Kauf". Wie fühlt sich der User? (Zufrieden).
     - Lernziel: User Satisfaction & Joy of Use.

4. "Scenario-Simulator"
   - Ziel: Teste das Produkt im Kontext.
   - Spielmechanik: 
     - "Der User sitzt in einer wackeligen Bahn mit schlechtem Netz". 
     - Funktioniert die App noch?
     - Lernziel: Kontextabhängige UX.

5. "UX-Metriken-Quiz"
   - Ziel: Miss den Erfolg von UX.
   - Spielmechanik: 
     - "Time on Task", "Success Rate", "SUS-Score".
     - Lernziel: Quantitative Bewertung von Erlebnisqualität.
