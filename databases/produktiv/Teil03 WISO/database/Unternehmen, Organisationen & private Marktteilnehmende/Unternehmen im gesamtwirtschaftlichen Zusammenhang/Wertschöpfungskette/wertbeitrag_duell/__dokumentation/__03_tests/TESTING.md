# Testing: Wertbeitrag-Duell

## Ziel
- Basisqualitaet sicherstellen: Spiel-Dateien vorhanden und Konfig ist lesbar.

## Struktur
- unit: lokale Smoke-Unit-Tests
- integration: spaetere Integrations-Tests
- fixtures: Testdaten
- artifacts: Screenshots/States
- reports: Testberichte

## Start
- Global + lokal: `node databases/produktiv/_testing/scripts/run-smoke-tests.mjs`
